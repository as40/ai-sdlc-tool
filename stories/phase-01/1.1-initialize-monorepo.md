# Story 1.1 — Initialize Monorepo & Engine Directories

## Phase
Phase 1: Foundation & Platform Scaffolding

## Status
`[ ] PENDING`

## Description
Scaffold the full monorepo structure with a Node.js/Express TypeScript backend and a React TypeScript frontend using pnpm workspaces. Create the physical SDLC engine directory structure inside the backend.

## Acceptance Criteria
- [ ] `pnpm-workspace.yaml` at the root referencing `packages/backend` and `packages/frontend`.
- [ ] `packages/backend` is a valid Node.js/TypeScript project with Express 5, Zod, Drizzle ORM, helmet, cors dependencies.
- [ ] `packages/frontend` is a valid React 18 + Vite + TypeScript + Tailwind CSS project.
- [ ] **shadcn/ui** is initialized in the frontend (`components.json` present, `src/lib/utils.ts` with `cn()` helper exists).
- [ ] Frontend `tailwind.config.ts` defines the project design tokens: zinc-950/900/800 backgrounds, indigo-500 accent, and CSS custom properties for dark mode matching `design/00-design-system.md`.
- [ ] Frontend has the full `src/` directory structure: `components/ui/`, `components/`, `pages/`, `hooks/`, `store/`, `utils/`.
- [ ] Frontend dependencies include: `lucide-react`, `react-resizable-panels`, `class-variance-authority`, `clsx`, `tailwind-merge`, `@radix-ui/react-slot`.
- [ ] `src/index.css` (global CSS) sets `@layer base` dark background (`bg-zinc-950`) and default text color so the app shell is dark mode by default.
- [ ] Root `package.json` has workspace-level scripts: `dev`, `build`, `test`, `lint`.
- [ ] The following SDLC engine directories exist inside `packages/backend/src/`:
  - `agents/`
  - `skills/`
  - `workflows/`
  - `prompts/`
  - `standards/`
  - `rules/`
  - `plugins/`
- [ ] Each engine directory has a `.gitkeep` placeholder and a `README.md` explaining its purpose.
- [ ] ESLint + Prettier configured at the root (shared config) and inherited by both packages.
- [ ] TypeScript `strict: true` in both `tsconfig.json` files.
- [ ] `.gitignore` excludes: `node_modules`, `dist`, `.env`, `*.local`, coverage reports.
- [ ] `.env.example` at the backend root documents all required environment variables (empty values).
- [ ] Backend starts without error: `pnpm --filter backend dev`.
- [ ] Frontend starts without error: `pnpm --filter frontend dev`.

## Technical Notes
- Use pnpm workspaces (not npm/yarn).
- Backend entry: `packages/backend/src/index.ts` — creates the Express app, mounts a health-check route `GET /health → 200 { status: 'ok' }`.
- Frontend entry: standard Vite React template with TypeScript.
- shadcn/ui setup: run `npx shadcn@latest init` with style=default, baseColor=zinc, cssVariables=yes. Do NOT install any individual components yet — that happens story-by-story in Phase 6+.
- `tailwind.config.ts` must extend colors with the tokens from `design/00-design-system.md`: `background: 'hsl(var(--background))'`, `foreground: 'hsl(var(--foreground))'`, etc. so shadcn components resolve correctly.
- `src/lib/utils.ts` must export `cn` using `clsx` + `tailwind-merge`.
- Path aliases: `@backend/*` → `src/*` in backend tsconfig; `@frontend/*` → `src/*` in frontend tsconfig; `@/*` → `src/*` in frontend (required by shadcn).
- Shared types package can be added later (Phase 2+); do not over-engineer now.
