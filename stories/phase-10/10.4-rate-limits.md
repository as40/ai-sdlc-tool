# Story 10.4 — Configurable Local Rate Limits

## Phase
Phase 10: Usage Audit & Analytics

## Dependencies
- Story 10.2, 2.3 must be `DONE`.

## Status
`[ ] PENDING`

## Description
Add a feature for team leads to set daily token or request limits per workspace, preventing autonomous loops from draining LLM credits.

## Acceptance Criteria
- [ ] `workspace_rate_limits` table: `id, workspace_id, daily_token_limit INT NULL, daily_request_limit INT NULL, alert_threshold_pct INT DEFAULT 80, created_at, updated_at`.
- [ ] `PUT /api/workspaces/:id/rate-limits` (WORKSPACE_OWNER+): set limits. NULL = unlimited.
- [ ] `LLMGateway` checks limits before each call:
  - Query today's token/request count from `token_usage_logs`.
  - If over limit: throw `RateLimitExceededError` (HTTP 429), halt the workflow.
  - If over alert threshold: emit WebSocket event `{ type: 'rate_limit_alert', payload: { pct, remaining } }`.
- [ ] UI: Rate Limits settings form in Workspace Settings.
- [ ] Alert banner in IDE when usage exceeds threshold %.

## Tests to Write
- Under limit → LLM call proceeds.
- Over daily token limit → 429, workflow halted.
- Alert threshold crossed → WS event emitted.
- NULL limit → unlimited (no check).

## Definition of Done
- Rate limits enforced before LLM calls. Alerts emitted. Settings UI works. Tests pass.
