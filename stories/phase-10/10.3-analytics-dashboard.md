# Story 10.3 — Workspace Admin Analytics Dashboard

## Phase

Phase 10: Usage Audit & Analytics

## Dependencies

- Story 10.2 must be `DONE`.

## Status

`[ ] PENDING`

## Description

Create a React analytics dashboard in Workspace Settings that shows token usage breakdown by agent, user, and model over configurable date ranges.

## Acceptance Criteria

- [ ] `GET /api/workspaces/:id/analytics/usage?from=&to=` (WORKSPACE_OWNER+):
  - Returns: `{ totalTokens, totalCost (estimated), byAgent: [...], byUser: [...], byModel: [...], dailyTrend: [...] }`.
  - Cost estimation: configurable price-per-1k-tokens per model in a constants file.
- [ ] Dashboard page in Workspace Settings → Analytics tab.
- [ ] Charts (using `recharts`):
  - Line chart: daily token usage trend.
  - Bar chart: usage by agent type.
  - Pie chart: usage by user.
- [ ] Date range picker (last 7 days / 30 days / custom).
- [ ] Summary cards: Total Tokens, Estimated Cost, Most Active Agent, Most Active User.
- [ ] CSV export button.
- [ ] CSV export: `GET /api/workspaces/:id/analytics/export?format=csv` returns a CSV file with `Content-Disposition: attachment; filename='analytics.csv'` header. Frontend 'Export CSV' button triggers a file download via `window.location.href = url`.
- [ ] Loading state: show skeleton rows while data fetches. Empty state: 'No activity yet' message when all chart data is zero.

## Technical Notes

- `recharts` is not in the CLAUDE.md tech stack list — add it as a confirmed frontend dependency (`pnpm --filter frontend add recharts`).
