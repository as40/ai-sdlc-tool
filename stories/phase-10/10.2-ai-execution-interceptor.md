# Story 10.2 — Backend AI Execution Interceptor

## Phase
Phase 10: Usage Audit & Analytics

## Dependencies
- Story 10.1, 5.4 must be `DONE`.

## Status
`[ ] PENDING`

## Description
Refactor the LLM Gateway to intercept every API response, extract usage metadata, and asynchronously insert a usage log record.

## Acceptance Criteria
- [ ] `LLMGateway.call()` method updated to extract `usage` from the API response.
- [ ] After every successful LLM call, asynchronously call `UsageLogRepository.insert()` with:
  - `workspaceId`, `userId`, `workflowExecutionId` (from context).
  - `model_name`, `provider`, `inputTokens`, `outputTokens`, `totalTokens`, `latencyMs`.
- [ ] Logging is **fire-and-forget** (non-blocking) — a failed insert must not affect the AI response.
- [ ] For local models: log `latencyMs` and `isLocal=true`; token counts set to 0 if unavailable.
- [ ] Error in usage logging: log to structured logger only, never surface to user.

## Files to Create

## Tests to Write
- After LLM call → usage log inserted with correct token counts.
- Usage log insert failure → LLM response still returned successfully.
- Local model call → `isLocal=true`, tokens=0.

## Definition of Done
- Every LLM API call produces a usage log. Logging never blocks responses. Tests pass.
