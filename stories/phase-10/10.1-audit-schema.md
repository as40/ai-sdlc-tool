# Story 10.1 — Execution Audit Database Schema

## Phase
Phase 10: Usage Audit & Analytics

## Dependencies
- Story 1.2 must be `DONE`.

## Status
`[ ] PENDING`

## Description
Create the `token_usage_logs` table to record every AI API call's usage metadata for BYOK tracking and cost attribution.

## Acceptance Criteria
- [ ] `token_usage_logs` table:
  | Column | Type |
  |--------|------|
  | id | UUID (v7) PK |
  | workspace_id | UUID FK → workspaces.id |
  | user_id | UUID FK → users.id |
  | workflow_execution_id | UUID FK → workflow_executions.id NULL |
  | agent_type | VARCHAR(100) |
  | model_name | VARCHAR(255) |
  | provider | VARCHAR(100) |
  | input_tokens | INTEGER |
  | output_tokens | INTEGER |
  | total_tokens | INTEGER |
  | latency_ms | INTEGER |
  | is_local | BOOLEAN DEFAULT false |
  | local_compute_ms | INTEGER NULL |
  | created_at | TIMESTAMPTZ DEFAULT NOW() |
- [ ] Indexes: `workspace_id`, `user_id`, `created_at` (for analytics queries).
- [ ] `UsageLogRepository` with: `insert(data)`, `sumByWorkspace(workspaceId, dateRange)`, `sumByUser(workspaceId, dateRange)`.
