# Story 10.5 — Org-Level Cost Governance & Budgets

## Phase

Phase 10: Usage Audit & Analytics

## Dependencies

- Story 10.1 must be `DONE` (`token_usage_logs` — the spend data source).
- Story 10.4 must be `DONE` (per-workspace rate limits — this generalises to org/team budgets).
- Story 2.4 should be `DONE` (workspaces/teams to allocate budgets and chargeback against).

## Status

`[ ] PENDING`

## Description

Story 10.4 caps per-workspace daily tokens; Story 5.4 caps per-workflow tokens. Enterprises also need **org- and team-level financial governance**: monthly budgets, hard quotas that halt work, spend alerts to finance, and cost allocation/chargeback so each team's AI spend is attributable. This is what a finance/procurement stakeholder asks for.

## Acceptance Criteria

### Budgets & Quotas

- [ ] `spend_budgets` table: `id, scope_type ('org'|'team'|'workspace'|'user'), scope_id, period ('daily'|'monthly'), token_limit BIGINT NULL, cost_limit_usd NUMERIC NULL, hard_stop BOOLEAN DEFAULT false, alert_thresholds_pct INT[] DEFAULT '{50,80,100}', created_at, updated_at`.
- [ ] Cost is derived from tokens via a configurable price book: `model_pricing` table `{ provider, model_name, input_price_per_1k, output_price_per_1k, effective_from }`. Local models can be assigned an amortised compute cost or zero.
- [ ] Budgets nest: a workflow's spend counts against its user, team, workspace, and org budgets simultaneously. The **most restrictive** applicable budget governs.
- [ ] `hard_stop: true` budget exhausted → the `LLMGateway` refuses new calls in scope with `BUDGET_EXCEEDED` (HTTP 429-class) and halts the workflow; `hard_stop: false` → warn-only, work continues.

### Alerts

- [ ] As cumulative spend crosses each `alert_thresholds_pct` mark, emit a `spend_alert` WS event and (if configured) an outbound notification (email/webhook) to the budget owner — deduplicated so each threshold fires once per period.
- [ ] Alert includes scope, period, current spend, limit, and projected end-of-period spend (simple run-rate extrapolation).

### Chargeback & Reporting

- [ ] `GET /api/orgs/:id/cost-report?period=...&groupBy=team|workspace|user|model` (org admin) — attributable spend breakdown for chargeback.
- [ ] Report is exportable as CSV for finance.
- [ ] `GET`/`PUT /api/orgs/:id/spend-budgets` and workspace-scoped equivalents — manage budgets at the level the caller is authorised for (org admin sets org/team; WORKSPACE_OWNER sets workspace/user within their workspace, bounded by the parent budget).

## Technical Notes

- Reuse the Story 10.4 pre-call check seam in `LLMGateway` — budgets are evaluated in the same gate as rate limits, so there is one place that can halt a call. Order: DLP (5.11) → rate limit (10.4) → budget (10.5) → provider.
- Spend aggregation must be efficient — pre-aggregate per-scope per-period counters (incremented on each `token_usage_logs` insert) rather than summing the raw log table on every call.
- Price book changes are versioned by `effective_from` so historical reports reflect the price at the time of use.
- Budgets set a **floor** that per-workspace and per-workflow caps cannot exceed — a workspace owner cannot grant themselves more than the org budget allows.

## Tests to Write

- Tokens logged → cost derived correctly from price book.
- Nested budgets → most restrictive governs.
- `hard_stop` budget exhausted → gateway refuses call, workflow halts.
- `hard_stop: false` exhausted → warn-only, call proceeds.
- Threshold crossings → one alert per threshold per period (dedup).
- Cost report groupBy team → attributable totals correct, no cross-tenant leakage.
- CSV export well-formed.
- Workspace owner cannot set a workspace budget above the parent org budget.
- Price book change with `effective_from` → historical report uses period-correct price.

## Definition of Done

- Org/team/workspace/user budgets enforced with hard-stop or warn semantics. Spend alerts fire once per threshold. Chargeback report attributes spend and exports for finance. Budgets nest with the most restrictive governing. Tests pass.
