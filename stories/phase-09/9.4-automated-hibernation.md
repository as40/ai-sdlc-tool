# Story 9.4 — Automated Resource Hibernation

## Phase
Phase 9: Containerized Sandbox Runtime Environment

## Dependencies
- Story 9.2 must be `DONE`.

## Status
`[ ] PENDING`

## Description
Implement a background cron job that monitors workspace activity and automatically stops sandbox containers after a period of inactivity to conserve resources.

## Acceptance Criteria
- [ ] `ActivityTracker` service records the last activity timestamp per workspace (in-memory + DB).
  - Activity events: file save, chat message, WebSocket ping from workspace user.
  - Updated via a lightweight `recordActivity(workspaceId)` call in relevant services.
- [ ] Hibernation cron runs every 2 minutes.
  - Checks each running sandbox: `NOW() - last_activity_at > HIBERNATION_TIMEOUT` (default 30 minutes, configurable via env).
  - If inactive: runs `docker-compose stop` for the workspace.
  - Updates sandbox status in DB to `HIBERNATED`.
  - Emits WebSocket event: `{ type: 'sandbox_hibernated', payload: { workspaceId } }`.
- [ ] IDE shows a "Sandbox Hibernating — Click to resume" banner when hibernated.
- [ ] Clicking the banner calls `POST .../sandbox/start` to wake the sandbox.
- [ ] `HIBERNATION_TIMEOUT_MINUTES` env var (default 30).
