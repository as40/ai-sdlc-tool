# Story 9.2 — Manual Sandbox Trigger & Proxy Setup

## Phase

Phase 9: Containerized Sandbox Runtime Environment

## Dependencies

- Story 9.1, 3.4 must be `DONE`.

## Status

`[ ] PENDING`

## Description

Add a "Start Sandbox" button in the IDE. On click, run docker-compose up and map container ports to a secure ephemeral URL via a reverse proxy for browser previewing.

## Acceptance Criteria

- [ ] `POST /api/workspaces/:id/sandbox/start` — starts the sandbox.
  - Checks env gateway status (story 3.4). Returns 423 if env keys missing.
  - **State-guarded (Story 9.6):** if sandbox status is `RUNNING`/`BOOTING`/`PREFLIGHT`, returns `409 SANDBOX_ALREADY_ACTIVE` — never reboots implicitly.
  - **Boot is explicit and optional:** only ever runs on this user action, never automatically on login, file save, or AI code changes.
  - If no `validated` boot recipe exists (Story 9.6), transitions to `AWAITING_BOOT_DECISION` and fires the "boot it / I'll set it up / no boot" HITL question instead of blindly running compose.
  - Runs migrations/seed from the boot recipe's `setup_commands` after `docker-compose up -d`, then waits on healthchecks before reporting ready — so the app boots into a **testable** state, not just "containers up".
  - Detects exposed ports from the compose file.
  - Creates an ephemeral reverse proxy route for each port using `http-proxy` middleware.
  - Returns `{ previewUrls: [{ service: string; url: string }] }`.
- [ ] `POST /api/workspaces/:id/sandbox/restart` — explicit, separate from start; requires `{ confirm: true }` (400 without). Stops then boots using the validated recipe. This is the **only** path that reboots a running sandbox.
- [ ] On boot failure, transitions to `MANUAL_SETUP` (Story 9.6) with logs surfaced — does not silently retry.
- [ ] `POST /api/workspaces/:id/sandbox/stop` — runs `docker-compose stop`.
- [ ] `GET /api/workspaces/:id/sandbox/status` — returns `{ running: boolean; services: [...] }`.
- [ ] "Start Sandbox" button in IDE TopNavBar. Shows spinner while starting.
- [ ] Preview URLs displayed as clickable links in IDE sidebar.
- [ ] Proxy URL format: `http://localhost:{proxyPort}/{workspaceId}/{service}/`.
- [ ] Start (`POST .../start`) and stop (`POST .../stop`) sandbox endpoints require `requireRole('DEVELOPER', 'WORKSPACE_OWNER', 'SUPER_ADMIN')`. `VIEWER` receives 403.

## Security Notes

- Proxy routes scoped to workspace ID — users can only access their own sandbox.
- `docker-compose` called with absolute path to compose file (no shell injection).

## Technical Notes

- Proxy URL `http://localhost:{proxyPort}` is only valid for same-host deployments. In Docker/networked environments, the proxy host must be configurable via `SANDBOX_PROXY_HOST` env var (default: `localhost`). Add `SANDBOX_PROXY_HOST` to `.env.example`.
