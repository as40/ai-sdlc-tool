# Story 9.3 — Live Terminal Logging Window

## Phase

Phase 9: Containerized Sandbox Runtime Environment

## Dependencies

- Story 9.2, 1.4 must be `DONE`.

## Status

`[ ] PENDING`

## Description

Create a collapsible log window at the base of the center workspace. Use WebSockets to pipe stdout and stderr streams from Docker containers directly into the UI.

## Acceptance Criteria

- [ ] Collapsible terminal log panel at the bottom of the Center Area.
- [ ] Panel height resizable by dragging the top edge.
- [ ] `GET /api/workspaces/:id/sandbox/logs?service=api` — streams Docker container logs via SSE or WebSocket.
  - Backend: `docker logs -f --tail 200 <container>` piped to WebSocket.
  - Each log line emitted as: `{ type: 'container_log', payload: { service, line, stream: 'stdout'|'stderr', timestamp } }`.
- [ ] Frontend renders log lines with:
  - Timestamp prefix.
  - Service name badge (colored per service).
  - Stderr lines in red.
- [ ] Service selector tabs if multiple containers running.
- [ ] "Clear" button clears the UI buffer (not the container logs).
- [ ] Auto-scroll to bottom on new logs (with "Pause scroll" button).
- [ ] Max 2000 lines in UI buffer (oldest dropped when exceeded).

## Technical Notes

- Use WebSocket (not SSE) for terminal log streaming, consistent with the rest of the architecture. Event type: `{ type: 'terminal_log', payload: { service: string; line: string; timestamp: string } }`. The 2000-line buffer limit is per-service, not total.
