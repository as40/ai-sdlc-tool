# Story 3.4 — Environment Variable Gateway

## Phase
Phase 3: Workspace Onboarding & Multi-Repo Universal Scanner

## Dependencies
- Story 3.2 must be `DONE`.

## Status
`[ ] PENDING`

## Description
Build a post-clone scanner that identifies required environment variables across all cloned repos (from `.env.example` files), then block sandbox execution until the user fills in the secrets via a secure UI form.

## Acceptance Criteria
- [ ] `EnvScanner` service runs after cloning completes.
- [ ] Scans all `.env.example`, `.env.template`, and `.env.sample` files across cloned repos.
- [ ] Collects all key names (ignoring values) into a deduplicated list per repo.
- [ ] `workspace_env_keys` table: `id, workspace_id, repo_id, key_name, value_encrypted, is_configured BOOLEAN, created_at, updated_at`.
- [ ] API:
  - `GET /api/workspaces/:id/env-keys` — list all required keys with `is_configured` status (never return values).
  - `POST /api/workspaces/:id/env-keys` — save/update values. Encrypts values with `CryptoService`.
  - `GET /api/workspaces/:id/env-keys/status` — returns `{ allConfigured: boolean; missing: string[] }`.
- [ ] Sandbox execution endpoint (story 9.2) checks `/env-keys/status` before starting containers. Returns `423 Locked` with missing keys if not all configured.
- [ ] Frontend: Environment Variables setup page with a form listing all required keys, each with a password-type input.

## Technical Notes
- Scan `.env.example` by reading line-by-line, extracting lines matching `KEY_NAME=` (strip value, keep key).
- Keys are WRITE-ONLY from the API — values are decrypted in memory only when writing `.env` files into the container (story 9.2).
- No plaintext env values ever returned over HTTP.

## Tests to Write
- `.env.example` with 3 keys → 3 records created in DB.
- Duplicate keys across repos → deduplicated per repo.
- GET status with unconfigured keys → `{ allConfigured: false, missing: [...] }`.
- POST saves encrypted value → GET does not return value.
- All keys configured → `{ allConfigured: true }`.

## Definition of Done
- All tests pass. Env gateway blocks sandbox start when keys are missing.
