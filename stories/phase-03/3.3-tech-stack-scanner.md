# Story 3.3 — Universal Tech-Stack Scanner

## Phase
Phase 3: Workspace Onboarding & Multi-Repo Universal Scanner

## Dependencies
- Story 3.2 must be `DONE`.

## Status
`[ ] PENDING`

## Description
Write an inspection engine that analyzes the cloned repositories to automatically detect the tech stack(s) in use. Store the detected platform attributes in the database for use by the AI prompt compiler and plugin loader.

## Acceptance Criteria
- [ ] `TechStackScanner` service scans each cloned repo directory.
- [ ] Detection rules (at minimum):
  | Signal File | Detected Stack |
  |---|---|
  | `package.json` (with react dep) | `react` |
  | `package.json` (with next dep) | `nextjs` |
  | `package.json` (with express dep) | `nodejs-express` |
  | `pom.xml` | `java-maven` |
  | `build.gradle` | `java-gradle` |
  | `requirements.txt` / `pyproject.toml` | `python` |
  | `go.mod` | `golang` |
  | `*.csproj` | `dotnet` |
  | `Cargo.toml` | `rust` |
  | `Gemfile` | `ruby` |
- [ ] Multiple stacks can be detected per repo (e.g., a monorepo with both Java and React).
- [ ] Results stored in `repositories.tech_stack` as JSONB: `{ primary: string; secondary: string[]; detected_at: ISO8601 }`.
- [ ] Scanner triggers automatically after successful clone (story 3.2 background worker calls it).
- [ ] `GET /api/workspaces/:id/repos/:repoId/tech-stack` — returns the stored JSONB.
- [ ] Frontend: displays detected stacks as badges on the repository cards in the workspace overview.

## Technical Notes
- Scanner reads files using `fs.promises` — never shells out.
- For `package.json`: parse JSON, check `dependencies` and `devDependencies` keys.
- Detection is additive — scan all signal files before writing the result.
- If no stack is detected: `{ primary: 'unknown', secondary: [] }`.

## Tests to Write
- `package.json` with React dep → detects `react`.
- `pom.xml` present → detects `java-maven`.
- Multiple signal files → multiple stacks in `secondary`.
- Empty directory → `{ primary: 'unknown' }`.
- Malformed `package.json` → gracefully returns `unknown`, logs warning.

## Definition of Done
- All tests pass. Stack data stored in DB. Badges visible in the UI.
