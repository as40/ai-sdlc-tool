# Story 13.3 — Custom MCP Schema Registration Component

## Phase
Phase 13: Advanced Workspace Customization & Validation Guardrails

## Dependencies
- Story 5.3, 13.1 must be `DONE`.

## Status
`[ ] PENDING`

## Description
Build a registry form for user-provided MCP servers that requires an executable command, environment variable mapping, and a valid JSON Schema describing the tool parameters.

## Acceptance Criteria
- [ ] `custom_mcp_servers` table: `id, workspace_id, name, command TEXT, env_vars JSONB, input_schema JSONB, created_at, updated_at`.
- [ ] Registration form fields:
  - **Name**: machine-readable slug (kebab-case, unique per workspace).
  - **Command**: shell command to start the MCP server (e.g., `node mcp-server.js`).
  - **Environment Variables**: key-value pairs.
  - **JSON Schema**: textarea with live JSON syntax validation.
- [ ] Frontend validates JSON Schema syntax before submission: `JSON.parse()` and check for required `type: 'object'` at root.
- [ ] `POST /api/workspaces/:id/mcp-servers` (WORKSPACE_OWNER+): saves custom MCP server.
  - Backend re-validates JSON Schema structure.
  - Returns 400 with details if schema is invalid.
- [ ] Registered custom MCPs are loaded by `RegistryLoader` alongside built-in skills.
- [ ] List view in Skills & MCPs tab showing all registered custom MCPs.
- [ ] Delete button removes custom MCP.

- Valid JSON Schema → saved.
- Invalid JSON → 400 with syntax error location.
- Missing `type: 'object'` → 400.
- Duplicate name → 409.
- DEVELOPER role → 403.
