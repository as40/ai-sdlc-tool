# Story 13.6 — Unbreakable Base System Prompt Compiler

## Phase

Phase 13: Advanced Workspace Customization & Validation Guardrails

## Dependencies

- Story 5.2, 13.5 must be `DONE`.

## Status

`[ ] PENDING`

## Description

Refactor the prompt constructor to implement a "sandwich" architecture: an immutable header and footer wrap the user's validated custom persona, ensuring global rules and formatting constraints can never be overridden.

## Acceptance Criteria

- [ ] `PromptCompiler.compile()` refactored to produce:
  ```
  [IMMUTABLE HEADER]
  {global rules, 90% test coverage enforcement, security mandates}

  [USER/WORKSPACE PERSONA]
  {validated custom agent persona or default}

  [PLUGIN STANDARDS]
  {tech-stack specific standards}

  [IMMUTABLE FOOTER]
  {JSON/Markdown formatting constraints, output format rules, tool usage requirements}
  ```
- [ ] `system-header.md` updated to include: test coverage ≥90%, no TODO without ticket, security OWASP rules, commit message standards.
- [ ] `system-footer.md` includes: output format rules (JSON schema for structured responses), mandated tool calls (log_feature_completion on COMPLETED), prohibited behaviors.
- [ ] Header and footer sourced from **encrypted assets** (story 11.2) — cannot be replaced by workspace config.
- [ ] Even if workspace persona is completely empty, header and footer are always present.
- [ ] Unit test: no matter what custom persona is provided, header and footer appear in the output.

## Technical Notes

- Dev-mode fallback: if Story 11.2 encrypted assets are not present (i.e., `ASSET_ENCRYPTION_KEY` env var not set or assets not bundled), the compiler falls back to loading header/footer from plaintext files at `packages/backend/src/prompts/header.md` and `packages/backend/src/prompts/footer.md`. Add these placeholder files with comment `<!-- Replace with encrypted asset in production -->`.
