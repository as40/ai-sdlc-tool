# Story 13.2 — Multi-Input Structural Agent Form with Zod Guardrails

## Phase
Phase 13: Advanced Workspace Customization & Validation Guardrails

## Dependencies
- Story 13.1, 5.1 must be `DONE`.

## Status
`[ ] PENDING`

## Description
Replace generic prompt textareas with structured multi-field agent forms validated by strict Zod schemas, enforcing minimum quality standards for agent prompts.

## Acceptance Criteria
- [ ] Agent configuration form with separate fields:
  - **Role** (min 50 chars): "You are a {role}..."
  - **Coding Standards** (min 200 chars): Framework-specific rules.
  - **Definition of Done** (min 100 chars): Criteria for task completion.
  - **Custom Instructions** (optional, max 500 chars).
- [ ] Zod schema enforces: minimum character lengths with descriptive error messages.
- [ ] Real-time validation: error shown inline as user types (on blur, not on every keystroke).
- [ ] Character counter displayed below each field.
- [ ] "Save" button disabled until all required validations pass.
- [ ] `PUT /api/workspaces/:id/agents/:agentName` — save custom agent persona (WORKSPACE_OWNER+).
  - Request validated with the same Zod schema on the backend.
  - Triggers Prompt Auditor (story 13.4) before committing to DB.
- [ ] `custom_agent_personas` table: `id, workspace_id, agent_name, role, coding_standards, definition_of_done, custom_instructions, created_at, updated_at`.
