# Story 13.1 — Tabbed Configuration Workspace Dashboard

## Phase

Phase 13: Advanced Workspace Customization & Validation Guardrails

## Dependencies

- Story 2.3 (RBAC), 2.4 (workspace) must be `DONE`.

## Status

`[ ] PENDING`

## Description

Create a comprehensive settings interface in React with five distinct sub-tabs, gated behind WORKSPACE_OWNER and SUPER_ADMIN roles.

## Acceptance Criteria

- [ ] Settings page at `/workspace/:id/settings`.
- [ ] Five tabs: **Integrations**, **Design Systems**, **Skills & MCPs**, **Agents**, **Workflows**.
- [ ] Access gate: DEVELOPER and VIEWER roles are redirected to `/unauthorized` (via `ProtectedRoute`).
- [ ] **Integrations tab**: Git provider, Jira, AI Providers (from existing stories).
- [ ] **Design Systems tab**: Figma API key config (story 7.4).
- [ ] **Skills & MCPs tab**: Custom MCP registration (story 13.3 — placeholder if not yet built).
- [ ] **Agents tab**: Agent persona customization (story 13.2 — placeholder if not yet built).
- [ ] **Workflows tab**: Workflow builder (story 14.1 — placeholder if not yet built).
- [ ] Tab state persisted in URL query param (`?tab=agents`).
- [ ] Breadcrumb navigation: Workspace > Settings > {Tab Name}.
- [ ] This story MIGRATES standalone settings pages created in earlier phases into tabs. The following existing pages are imported as tab content (not rebuilt from scratch): AI Config (Story 2.5 → `AIConfigPage`), Git Auth (Story 3.1 → `GitSettingsPanel`), Env Vars (Story 3.4 → `EnvVarSettingsPanel`), Jira (Story 7.1 → `JiraSettingsPanel`), Webhooks (Story 8.4 → `WebhookSettingsPanel`). Agents (new tab), SSO (Story 2.1 → `SSOConfigForm`).
- [ ] Default tab when no `?tab=` query param: `ai-config`.
- [ ] Standalone routes from earlier stories (`/workspaces/:id/settings/ai-config`, `/workspaces/:id/settings/git`, etc.) remain in `App.tsx` as redirects to `/workspaces/:id/settings?tab=<name>` for backwards compatibility.
