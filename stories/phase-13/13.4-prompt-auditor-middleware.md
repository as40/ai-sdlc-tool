Phase 13: Advanced Workspace Customization & Validation Guardrails

## Status
`[ ] PENDING`

## Description
Implement an asynchronous backend evaluation worker that audits user-submitted agent prompts for adherence to core engineering principles (SOLID, DRY, documentation standards) before committing them to the database.

## Acceptance Criteria
- [ ] `PromptAuditorService.audit(promptText: string): Promise<AuditResult>`:
  - Calls a low-cost LLM with the prompt text and an auditing meta-prompt.
  - Auditing meta-prompt checks for: SOLID principles referenced, DRY principle mentioned, documentation requirements specified, security guidelines, testing requirements.
  - Returns `{ passed: boolean; score: number (0-100); deficiencies: string[] }`.
- [ ] If `passed: false`: `AgentPersonaService` cancels the DB transaction and returns `422 Unprocessable Entity` with the deficiency list.
- [ ] If `passed: true`: proceed with saving.
- [ ] Minimum passing score: 70 (configurable via env `PROMPT_AUDIT_MIN_SCORE`).
- [ ] Audit result logged to a new `prompt_audit_logs` table for admin review.
- [ ] Frontend: displays deficiency list inline when save returns 422.
- [ ] Auditor can be bypassed by SUPER_ADMIN only (via `?skipAudit=true` query param).

## Tests to Write
- High-quality prompt → passed, saved to DB.
- Vague prompt with no standards → 422 with deficiencies.
- SUPER_ADMIN with skipAudit → bypasses auditor.
- Auditor LLM failure → graceful degradation (log error, allow save with warning).

## Definition of Done
- Poor prompts rejected with deficiency details. Audit logged. SUPER_ADMIN bypass works. Tests pass.
