## Description
Build a `configResolver` utility that handles the configuration cascade: merging system defaults with workspace-level overrides, custom MCPs, and agent persona customizations.

## Acceptance Criteria
- [ ] `ConfigResolver.resolve(workspaceId: string): Promise<ResolvedConfig>`:
  - Loads: system default agents (from registry), system default skills, system default plugins.
  - Loads: workspace custom agent personas (DB).
  - Loads: workspace custom MCP servers (DB).
  - Merges according to rules:
    - Custom agent persona: spread-merges on top of default: `{ ...defaultAgent, ...customPersona }`.
    - Custom MCPs: appended to built-in skill list (not replacing).
    - Missing custom config: falls back to system default (never drops a default).
- [ ] `ResolvedConfig` shape: `{ agents: AgentDefinition[]; skills: SkillDefinition[]; plugins: PluginDefinition[] }`.
- [ ] `ConfigResolver` cached per workspace (invalidated when workspace config changes).
- [ ] `WorkflowOrchestrator` calls `ConfigResolver.resolve()` at the start of each workflow.
- [ ] Fallback guarantee: if workspace has no customizations, behavior is identical to system defaults.
