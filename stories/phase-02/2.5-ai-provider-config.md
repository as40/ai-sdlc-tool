# Story 2.5 — Universal AI Provider Configuration

## Phase
Phase 2: Authentication & Secure Configuration Setup

## Dependencies
- Story 1.3 (CryptoService), Story 2.3 (RBAC) must be `DONE`.

## Status
`[ ] PENDING`

## Description
Build a secure form and backend API for `WORKSPACE_OWNER` or `SUPER_ADMIN` to define AI model configurations — either cloud BYOK (Bring Your Own Key) or local/self-hosted models. API keys are always stored encrypted.

## Acceptance Criteria
- [ ] `POST /api/workspaces/:id/ai-config` — create AI provider config (WORKSPACE_OWNER+ only).
  - Request: `{ provider: string; modelName: string; apiKey?: string; baseUrl?: string; isLocal: boolean }`.
  - `apiKey` encrypted with `CryptoService` before DB insert; stored in `ai_configurations.api_key_encrypted`.
  - Response: config object with `apiKey` field omitted (never returned to client).
- [ ] `GET /api/workspaces/:id/ai-config` — list all configs (WORKSPACE_OWNER+ only). Never returns the raw key.
- [ ] `PUT /api/workspaces/:id/ai-config/:configId` — update config.
- [ ] `DELETE /api/workspaces/:id/ai-config/:configId` — delete config.
- [ ] `POST /api/workspaces/:id/ai-config/:configId/test` — test the connection by making a minimal API call (e.g., listing models or sending a 1-token prompt). Returns `{ success: boolean; latencyMs: number; error?: string }`.
- [ ] Frontend form in Workspace Settings with fields:
  - Provider dropdown (Anthropic, OpenAI, Azure OpenAI, Custom/Local).
  - Model name input.
  - API key input (masked, write-only).
  - Base URL input (for local/Azure).
  - Local toggle.
  - "Test Connection" button.
- [ ] Multiple AI configs can exist per workspace (e.g., one for code generation, one for embeddings).

## Technical Notes
- `apiKey` is WRITE-ONLY from the API's perspective — the decrypted value is used internally by the `LLMGateway` service (Phase 5) but never returned over HTTP.
- Zod schema must enforce: if `isLocal=false`, `apiKey` is required.
- `LLMGateway` service stub created here — actual implementation in Phase 5.
