## Description

Implement enterprise SSO login via OpenID Connect (OIDC) and SAML 2.0 using `passport.js`. Build an admin UI for clients to configure their Okta or Azure AD provider settings.

## Acceptance Criteria

- [ ] `passport-openidconnect` strategy registered for OIDC login flow.
- [ ] `passport-saml` strategy registered for SAML 2.0 login flow.
- [ ] Routes:
  - `GET /api/auth/oidc` ‚Äî initiates OIDC redirect.
  - `GET /api/auth/oidc/callback` ‚Äî handles OIDC callback, issues JWT, redirects to frontend.
  - `POST /api/auth/saml` ‚Äî initiates SAML redirect.
  - `POST /api/auth/saml/callback` ‚Äî handles SAML assertion, issues JWT, redirects to frontend.
- [ ] Admin UI page (accessible only to `SUPER_ADMIN`) at `/settings/sso`:
  - OIDC fields: Client ID, Client Secret, Issuer URL, Redirect URI.
  - SAML fields: Entry Point URL, Certificate (PEM), Issuer.
  - "Save" button stores config encrypted (using `CryptoService` from 1.3) in a new `sso_configurations` table.
  - "Test Connection" button initiates a login flow in a popup.
- [ ] SSO config is stored per-workspace in the `sso_configurations` table.
- [ ] On successful SSO login, JIT provisioning middleware is triggered (story 2.2 hook point).
- [ ] Session state for the OAuth flow stored with `express-session` (in-memory for dev, Redis for prod).
- [ ] OIDC/SAML callback (`GET /api/auth/oidc/callback`) redirects to `{FRONTEND_URL}/?token={jwt}`. The frontend `HomePage` reads the `?token=` query param on mount, calls `setToken()`, then proceeds with the workspace-fetch-and-redirect flow (same as Story 2.0).
- [ ] `SSOSettingsPage.tsx` is mounted at route `/workspaces/:id/settings/sso` in `App.tsx` as a standalone route (migrated into the Settings tab shell in Story 13.1). Requires `ProtectedRoute` with `WORKSPACE_OWNER` role.

## Technical Notes

- Passport.js is the auth abstraction layer ‚Äî strategies are plugins.
- OIDC callback: extract `profile.email`, call JIT service, issue JWT.
- SAML callback: same flow.
- Never store the Client Secret in plaintext ‚Äî encrypt with `CryptoService`.
- `sso_configurations` table: `id, workspace_id, provider ('oidc'|'saml'), config_encrypted TEXT, created_at, updated_at`.
- **Redis session store:** Production Redis setup has no dedicated story. For Phase 2, `express-session` uses `MemoryStore` in all environments. Add `redis` + `connect-redis` when production deployment is configured (Story 11.5).
