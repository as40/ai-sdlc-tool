# Story 2.4 — Workspace Creation & Team Invites

## Phase

Phase 2: Authentication & Secure Configuration Setup

## Dependencies

- Story 2.3 must be `DONE`.

## Status

`[ ] PENDING`

## Description

Build the UI and backend for a `WORKSPACE_OWNER` to create a new workspace and invite other provisioned users to join with specific role permissions.

## Acceptance Criteria

- [ ] `POST /api/workspaces` — creates a new workspace (requires auth, any role).
  - Request: `{ name: string }`.
  - Response: the created workspace object.
  - Creator is automatically added to `workspace_members` as `WORKSPACE_OWNER`.
- [ ] `GET /api/workspaces` — lists all workspaces the authenticated user is a member of.
- [ ] `GET /api/workspaces/:id` — get workspace detail (members only).
- [ ] `POST /api/workspaces/:id/invites` — invite a user to the workspace.
  - Requires `WORKSPACE_OWNER` or `SUPER_ADMIN`.
  - Request: `{ email: string; role: 'DEVELOPER' | 'VIEWER' }`.
  - Looks up the user by email in `users` table. If not found: returns 404 (user must have logged in once via SSO first).
  - Adds user to `workspace_members` if not already a member.
- [ ] `DELETE /api/workspaces/:id/members/:userId` — remove a member.
  - Requires `WORKSPACE_OWNER` or `SUPER_ADMIN`.
  - Cannot remove yourself (owner).
- [ ] Frontend Workspace Creation page with a simple form.
- [ ] Frontend Team Management page showing current members with their roles and an "Invite" form.
- [ ] `WorkspacePage.tsx` is mounted at route `/workspaces/new` in `App.tsx`. Requires `ProtectedRoute` with minimum `DEVELOPER` role. On successful creation, navigates to `/workspaces/:id/settings/ai-config`.
- [ ] `TeamPage.tsx` is mounted at route `/workspaces/:id/team` in `App.tsx`. Requires `ProtectedRoute` with minimum `DEVELOPER` role.
- [ ] All form states covered: loading (button shows `'Creating…'`/`'Inviting…'`), error (shows `<Alert variant="error">` with server `detail`), success (navigates or shows `<Alert variant="success">`).

## Technical Notes

- All queries filter by `workspace_id` from `req.workspaceId` (set by workspace membership guard middleware).
- Workspace membership guard: `workspaceGuard` middleware checks `workspace_members` for the authenticated user and attaches `req.workspaceId` and `req.workspaceMember`.
- Use transactions for invite creation to avoid partial state.

- Create workspace → creator becomes WORKSPACE_OWNER in members table.

- Cannot remove yourself → 400.
