# Story 2.0 — Local Development Mock Authentication

## Phase
Phase 2: Authentication & Secure Configuration Setup

## Dependencies
- Story 1.1, 1.2 must be `DONE`.

## Status
`[ ] PENDING`

## Description
Build a local-dev-only mock authentication system so engineers can develop and test RBAC-protected features without a real SSO/IdP. This must be completely disabled in production.

## Acceptance Criteria
- [ ] Express route `POST /api/auth/mock-login` exists **only when `NODE_ENV=development`**. In production/test, the route returns `404`.
- [ ] Request body: `{ role: 'SUPER_ADMIN' | 'WORKSPACE_OWNER' | 'DEVELOPER' | 'VIEWER', email?: string }`.
- [ ] The route looks up (or creates) a user with the given email in the `users` table and returns a signed JWT with the requested role.
- [ ] JWT payload contains: `{ sub: userId, email, role, workspaceId?: string, iat, exp }`.
- [ ] JWT signed with `JWT_SECRET` env var; expiry 8 hours (long for local dev comfort).
- [ ] A React dev panel component appears on the login screen **only when `VITE_ENV=development`**.
- [ ] The dev panel shows 4 buttons: one per role. Clicking a button calls the mock-login endpoint and stores the JWT in `localStorage`.
- [ ] After login, the app redirects to the main workspace dashboard.
- [ ] The dev panel is tree-shaken out of production builds entirely (conditional import via `import.meta.env.DEV`).
- [ ] `requireAuth` middleware (used in later stories) validates the JWT produced by this route identically to real SSO tokens.

## Technical Notes
- Gate the route: `if (process.env.NODE_ENV !== 'development') return next()` before registering.
- Use `jsonwebtoken` for signing/verifying.
- `requireAuth` middleware: extract Bearer token, `jwt.verify(token, JWT_SECRET)`, attach `req.user` to the request.
- The dev panel is a purely visual dev tool — no tests required for the panel itself, but the `/api/auth/mock-login` endpoint must be fully tested.

## Tests to Write
### `auth.mock-login.test.ts`
- Returns 404 when `NODE_ENV=production`.
- Returns 400 for invalid role (Zod validation).
- Returns 200 with a valid JWT for each of the 4 roles.
- JWT payload contains the correct role.
- JWT is verifiable with `JWT_SECRET`.

### `require-auth.test.ts`
- Returns 401 with no Authorization header.
- Returns 401 with a malformed token.
- Returns 401 with an expired token.
- Attaches `req.user` and calls `next()` for a valid token.

## Definition of Done
- All tests pass.
- Dev panel renders in development mode and successfully logs in as each role.
- Production mode: `/api/auth/mock-login` returns 404; dev panel is absent from bundle.
