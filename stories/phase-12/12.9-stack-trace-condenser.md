# Story 12.9 — Context Stack Trace Condenser

## Phase
Phase 12: Session Isolation & Memory Lifecycle

## Dependencies
- Story 12.6 must be `DONE`.

## Status
`[ ] PENDING`

## Description
Implement an automatic cleanup hook that intercepts terminal outputs containing long stack traces, truncating repetitive mid-sections while preserving the core error boundaries before the next LLM call.

## Acceptance Criteria
- [ ] `StackTraceCondenser.condense(message: string): string`:
  - Input: a terminal log or AI message string.
  - If length > 2000 characters AND contains a stack trace signature (`at `, `Error:`, `Traceback`):
    - Keep first 800 characters (error header + initial frames).
    - Replace the middle with `\n[... N lines truncated ...]\n`.
    - Keep last 400 characters (final frames + resolution hints).
    - Return the condensed string.
  - If length ≤ 2000 or no stack trace signature: return unchanged.
- [ ] Hook applied to all `tool` role messages in `chat_messages` before they're included in the next LLM call context.
- [ ] Hook applied in the chat response before storing terminal command output to DB.
- [ ] Original full output stored in DB if needed for human review — only the condensed version passed to LLM.

## Tests to Write
- Short message → returned unchanged.
- Long message without stack trace → returned unchanged.
- Long message with stack trace → middle truncated, header + tail preserved.
- Exact character boundaries correct.
- LLM receives condensed version; DB stores original.

## Definition of Done
- Long stack traces condensed. LLM context reduced. Original preserved in DB. Tests pass.
