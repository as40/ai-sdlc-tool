# Story 12.5 — Background Summarization Pipeline

## Phase
Phase 12: Session Isolation & Memory Lifecycle

## Dependencies
- Story 12.1, 5.4 must be `DONE`.

## Status
`[ ] PENDING`

## Description
Build an Express utility that takes an array of chat messages, sends them to a low-cost LLM for summarization, and replaces the array with a single condensed context block.

## Acceptance Criteria
- [ ] `SummarizationService.summarize(sessionId: string): Promise<string>`:
  - Fetches all messages for the session from DB.
  - Builds a summarization prompt: "Summarize the following conversation, preserving: decisions made, code changes described, errors encountered, and current task state."
  - Calls a low-cost model (configurable — e.g., `claude-haiku-4-5` or workspace's economy model).
  - Returns the summary string.
- [ ] After summarization:
  - Deletes all existing messages for the session.
  - Inserts a single `system` message with role `system` and content = the summary.
  - Inserts a system chip message: `"Context compacted — {N} messages summarized"`.
  - Updates `chat_sessions.status = 'COMPACTED'` and resets `token_count`.
- [ ] WebSocket event: `{ type: 'context_compacted', payload: { sessionId, messageCount } }`.
- [ ] Used by: `/compact` slash command (6.8) and auto-compaction middleware (12.6).
