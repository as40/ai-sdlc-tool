# Story 12.2 — The Hard Purge UI Hook

## Phase

Phase 12: Session Isolation & Memory Lifecycle

## Dependencies

- Story 12.1, 7.3 must be `DONE`.

## Status

`[ ] PENDING`

## Description

Build the React hook that triggers when a user switches active Jira stories — force-clearing the local chat state, resetting the Git Diff viewer, and requesting a fresh session ID.

## Acceptance Criteria

- [ ] `useJiraStorySwitch` hook:
  - Called when user selects a different Jira story in the sidebar.
  - 1. Clears local Zustand chat state.
  - 2. Resets Git Diff viewer state (story 6.5).
  - 3. Calls `POST /api/sessions/new?jiraStoryId=PROJ-42` to get a fresh `session_id`.
  - 4. Updates active `session_id` in the editor store.
  - 5. Attaches new Jira story context (story 7.3 pipeline).
- [ ] Confirmation dialog if the current session has a RUNNING workflow: "A workflow is running. Switching will cancel it. Continue?".
- [ ] Confirmation dialog when active workflow detected uses shadcn `AlertDialog`. Title: 'Session has an active workflow', description: 'A workflow is currently running. Ending this session will abandon the workflow. Continue?', confirm button variant: `destructive`.
- [ ] Old session archived (`status = 'ARCHIVED'`) in DB — not deleted (for audit trail).
- [ ] New session inherits no messages from the old session.

## Technical Notes

- Session creation uses JSON body, not query params. Change `POST /api/sessions/new?jiraStoryId=` to `POST /api/sessions` with body `{ jiraStoryId?: string }` for consistency with the rest of the API.
