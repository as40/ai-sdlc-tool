## Description

Create a `log_feature_completion` MCP skill that the AI is mandated to call when a workflow reaches COMPLETED, automatically appending a summary to `STORY_TRACKER.md`.

## Acceptance Criteria

- [ ] `log_feature_completion` skill:
  - Input:
    ```typescript
    {
      jiraStoryId: string;
      summary: string;
      filesModified: string[];
      testsPassed: boolean;
      patternsDiscovered?: Array<{ key: string; content: string }>;  // optional
    }
    ```
  - Appends a formatted entry to `STORY_TRACKER.md` at the workspace root:
    ```
    ## [PROJ-42] вҖ” 2026-06-30
    **Summary**: {summary}
    **Files**: {filesModified}
    **Tests**: {testsPassed ? 'вң“' : 'вң—'}
    ```
  - Creates `STORY_TRACKER.md` if it doesn't exist.
  - Returns `{ appended: true; path: string }`.
  - If `patternsDiscovered` is provided and non-empty: for each entry, calls `MemoryRepository.upsert()` with `type: 'pattern'`, `key`, `content`, `confidence: 1.0` (human-validated via COMPLETED state). This is the primary mechanism for agents to permanently teach the system a new workspace pattern.
- [ ] The `WorkflowOrchestrator` system prompt mandates calling this skill when transitioning to COMPLETED. The prompt instructs: "Before calling `log_feature_completion`, reflect on whether you discovered any reusable patterns specific to this codebase that would help future tasks. If yes, include them in `patternsDiscovered`."
- [ ] Skill validated: `jiraStoryId` must match `[A-Z]+-\d+` pattern.
- [ ] `patternsDiscovered[].key` must match `[a-z0-9-]+` (kebab-case, max 100 chars). Reject non-conforming keys with `VALIDATION_ERROR`.
- [ ] Skill appends atomically (uses file lock or sequential write queue to prevent corruption from concurrent workflows).

## Tests to Write

- Valid call вҶ’ entry appended to file.
- File doesn't exist вҶ’ created first.
- Invalid jiraStoryId format вҶ’ validation error.
- Concurrent calls вҶ’ no file corruption (sequential write test).
- `patternsDiscovered` provided вҶ’ each pattern upserted to `workspace_memory` with `confidence: 1.0`.
- `patternsDiscovered` empty вҶ’ no memory writes, no error.
- Pattern key with uppercase or spaces вҶ’ `VALIDATION_ERROR`.
- Memory upsert fails вҶ’ warning logged, file append still completes (memory write is non-critical).

## Definition of Done

- Skill appends correctly. Concurrent writes safe. Format matches spec. Tests pass.
