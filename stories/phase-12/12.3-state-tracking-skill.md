# Story 12.3 — Automated State-Tracking Skill

## Phase
Phase 12: Session Isolation & Memory Lifecycle

## Dependencies
- Story 5.3, 5.4 must be `DONE`.

## Status
`[ ] PENDING`

## Description
Create a `log_feature_completion` MCP skill that the AI is mandated to call when a workflow reaches COMPLETED, automatically appending a summary to `STORY_TRACKER.md`.

## Acceptance Criteria
- [ ] `log_feature_completion` skill:
  - Input: `{ jiraStoryId: string; summary: string; filesModified: string[]; testsPassed: boolean }`.
  - Appends a formatted entry to `STORY_TRACKER.md` at the workspace root:
    ```
    ## [PROJ-42] — 2026-06-30
    **Summary**: {summary}
    **Files**: {filesModified}
    **Tests**: {testsPassed ? '✓' : '✗'}
    ```
  - Creates `STORY_TRACKER.md` if it doesn't exist.
  - Returns `{ appended: true; path: string }`.
- [ ] The `WorkflowOrchestrator` system prompt mandates calling this skill when transitioning to COMPLETED.
- [ ] Skill validated: `jiraStoryId` must match `[A-Z]+-\d+` pattern.
- [ ] Skill appends atomically (uses file lock or sequential write queue to prevent corruption from concurrent workflows).

## Files to Create
```
packages/backend/src/skills/log-feature-completion/
├── skill.config.json
├── log-feature-completion.skill.ts
└── __tests__/
    └── log-feature-completion.test.ts
```

## Tests to Write
- Valid call → entry appended to file.
- File doesn't exist → created first.
- Invalid jiraStoryId format → validation error.
- Concurrent calls → no file corruption (sequential write test).

## Definition of Done
- Skill appends correctly. Concurrent writes safe. Format matches spec. Tests pass.
