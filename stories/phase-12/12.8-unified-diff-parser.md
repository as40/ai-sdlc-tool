# Story 12.8 — Unified Diff Return Parser

## Phase

Phase 12: Session Isolation & Memory Lifecycle

## Dependencies

- Story 5.3 (skill router), 5.4 must be `DONE`.

## Status

`[ ] PENDING`

## Description

Configure the Developer Agent to return code modifications exclusively in Unified Diff format and implement a server-side patching utility that applies these diffs to the codebase.

## Acceptance Criteria

- [ ] Developer Agent system prompt updated to mandate: "Return all code changes as a Unified Diff (GNU diff format). Never return full file contents."
- [ ] `DiffPatchService.apply(repoPath: string, unifiedDiff: string): Promise<PatchResult>`:
  - Parses the unified diff string.
  - Validates it is well-formed (correct `---`/`+++` headers, `@@` hunks).
  - Applies the diff to the target file(s) using `patch` library or Node.js implementation.
  - Returns `{ success: boolean; filesPatched: string[]; errors: string[] }`.
- [ ] New MCP skill: `apply_diff(unifiedDiff: string)` — calls `DiffPatchService.apply()`.
- [ ] If the diff fails to apply (e.g., context mismatch): return a detailed error to the AI, asking it to regenerate with correct context.
- [ ] Full-file code generation by the AI is intercepted and rejected: if the AI returns a code block > 500 lines, prompt it to re-format as a diff.

## Technical Notes

- Full-file interception point: in the Orchestrator Supervisor (Story 5.4), after receiving an LLM response and before passing it to the skill router, check if the response contains a code block > 500 lines. If so, inject a follow-up system message: 'Your response contained a full file rewrite (>500 lines). Please reformat as a unified diff (`--- a/file` / `+++ b/file` / `@@ ... @@` format) showing only the changed lines.' The check lives in `src/agents/supervisor/response-interceptor.ts`.
