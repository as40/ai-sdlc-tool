# Story 12.7 — AST Snippet Isolation Middleware

## Phase
Phase 12: Session Isolation & Memory Lifecycle

- Story 4.2 (AST chunking), 5.3 (skill router) must be `DONE`.

## Status
`[ ] PENDING`

## Description
Build an AST-based file extractor that isolates targeted functions or classes from source files based on vector similarity matches, passing only line-numbered snippets to the LLM instead of full files.

## Acceptance Criteria
- [ ] `AstSnippetExtractor` service:
  - Input: `{ filePath: string; targetSymbol: string; contextLines: number (default 5) }`.
  - Reads the file, parses it with the appropriate AST parser.
  - Locates the target function/class by name.
  - Returns a string containing only the matched symbol with `contextLines` of surrounding context, each line prefixed with its line number: `"42: function foo() {"`
- [ ] Updated `search_codebase` skill: after vector similarity search, calls `AstSnippetExtractor` on each matching chunk's source file/symbol.
- [ ] Full file bodies are NEVER passed to the LLM when a symbol match is found.
- [ ] Fallback: if AST parsing fails for a file type, pass the raw chunk content (not the full file).
- [ ] Line numbers included in all extracted snippets.

## Definition of Done
