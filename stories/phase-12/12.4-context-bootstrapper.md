# Story 12.4 — Context Bootstrapper Middleware

## Phase

Phase 12: Session Isolation & Memory Lifecycle

## Dependencies

- Story 12.1, 7.1 must be `DONE` (12.1 defines the `workflow_errors` table queried for error memory).
- Story 4.4 should be `DONE` for workspace memory sections; if not, those sections are silently omitted.

## Status

`[ ] PENDING`

## Description

Build Express middleware that triggers when a new session starts, automatically compiling the Jira story data, root rule files, and tech stack info into a clean initial system message for the Orchestrator.

## Acceptance Criteria

- [ ] `contextBootstrap` middleware runs on `POST /api/sessions/new`.
- [ ] Compiles an initial system message containing sections in this order:
  1. Workspace rules from `src/rules/` files (concatenated).
  2. Active Jira story detail (title, description, acceptance criteria, subtasks).
  3. Workspace tech stack summary (from DB).
  4. Current repository structure (top-level directories).
  5. **Workspace patterns** (if Story 4.4 is available): queries `MemoryRepository.findByWorkspaceAndType(workspaceId, 'pattern')`. If records exist, includes:
     ```
     WORKSPACE PATTERNS (learned from past successful workflows):
     - {key}: {content}
     ```
     Omit entirely if none exist.
  6. **Workspace constraints** (if Story 4.4 is available): queries `MemoryRepository.findByWorkspaceAndType(workspaceId, 'constraint')` and `findByWorkspaceAndType(workspaceId, 'architectural_decision')`. If records exist, includes:
     ```
     ARCHITECTURAL CONSTRAINTS (always follow these):
     - {content}
     ```
     Omit entirely if none exist.
  7. **Workspace preferences** (populated by Story 12.11): queries `MemoryRepository.findByWorkspaceAndType(workspaceId, 'preference')`, filters to `confidence >= 0.5`. If records exist, includes:
     ```
     WORKSPACE PREFERENCES (learned from team feedback — follow unless the story requires otherwise):
     - {content}
     ```
     Omit entirely if none exist.
  8. **Error memory block** (only if a `jira_story_id` is present): queries `workflow_errors` table via `WorkflowErrorRepository.findByJiraStoryId()` for any prior failures on this story. If records exist, includes:
     ```
     ⚠️ PRIOR FAILURE MEMORY for {JIRA-KEY}:
     The following tool calls failed in previous sessions on this story and should be avoided:
     - {tool_name} (error: {error_type}, resolution: {resolution})
     Do not repeat these patterns without a materially different approach.
     ```
     Omit entirely if no prior errors exist.
  9. Instruction: "You are working on story {JIRA-KEY}. Follow the rules and memories above."
- [ ] Each memory section query that fails (DB error) is handled gracefully: log a warning, omit that section, continue building the rest of the bootstrap. No memory section is blocking.
- [ ] This compiled message inserted as the first `system` message in `chat_messages` for the new session.
- [ ] Session `token_count` initialized to the token count of this bootstrap message.
- [ ] Bootstrap message is re-built fresh for every new session (never reused from a prior session).
- [ ] If no Jira story attached: bootstrap only includes workspace rules and stack info.

## Technical Notes

- `src/rules/` files referenced as workspace rules are the files created in Story 1.1 during monorepo init (`.rules/general.md`, `.rules/security.md`, `.rules/testing.md`, `.rules/architecture.md` at the repo root). The bootstrapper reads these files from the filesystem at bootstrap time.

## Tests to Write

- With Jira story → bootstrap includes story detail.
- Without Jira story → bootstrap includes rules only.
- Bootstrap message is first message in session.
- Token count initialized correctly.
- Pattern memories exist → `WORKSPACE PATTERNS` section included.
- No pattern memories → section omitted.
- Constraint/architectural_decision memories exist → `ARCHITECTURAL CONSTRAINTS` section included.
- Preference memories with `confidence >= 0.5` exist → `WORKSPACE PREFERENCES` section included.
- Preference memories all below `confidence 0.5` → section omitted.
- Prior `workflow_errors` exist for story → error memory block included.
- No prior errors for story → error memory block omitted.
- Any single memory query fails (DB error) → warning logged, that section omitted, rest of bootstrap continues.

## Definition of Done

- Every new session starts with a comprehensive, structured context. Tests pass.
