# Story 12.4 — Context Bootstrapper Middleware

## Phase

Phase 12: Session Isolation & Memory Lifecycle

## Dependencies

- Story 12.1, 7.1 must be `DONE`.

## Status

`[ ] PENDING`

## Description

Build Express middleware that triggers when a new session starts, automatically compiling the Jira story data, root rule files, and tech stack info into a clean initial system message for the Orchestrator.

## Acceptance Criteria

- [ ] `contextBootstrap` middleware runs on `POST /api/sessions/new`.
- [ ] Compiles an initial system message containing:
  1. Workspace rules from `src/rules/` files (concatenated).
  2. Active Jira story detail (title, description, acceptance criteria, subtasks).
  3. Workspace tech stack summary (from DB).
  4. Current repository structure (top-level directories).
  5. Instruction: "You are working on story {JIRA-KEY}. Follow the rules above."
- [ ] This compiled message inserted as the first `system` message in `chat_messages` for the new session.
- [ ] Session `token_count` initialized to the token count of this bootstrap message.
- [ ] Bootstrap message is re-built fresh for every new session (never reused from a prior session).
- [ ] If no Jira story attached: bootstrap only includes workspace rules and stack info.

## Technical Notes

- `src/rules/` files referenced as workspace rules are the files created in Story 1.1 during monorepo init (`.rules/general.md`, `.rules/security.md`, `.rules/testing.md`, `.rules/architecture.md` at the repo root). The bootstrapper reads these files from the filesystem at bootstrap time.
