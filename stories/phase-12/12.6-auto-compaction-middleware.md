# Story 12.6 — Auto-Compaction Middleware

## Phase
Phase 12: Session Isolation & Memory Lifecycle

## Dependencies
- Story 12.5 must be `DONE`.

## Status
`[ ] PENDING`

## Description
Implement token-monitoring middleware that automatically routes a session through the summarization pipeline before responding if the token count exceeds a threshold.

## Acceptance Criteria
- [ ] `autoCompactionMiddleware` runs before every `POST /api/chat` request.
- [ ] Reads `chat_sessions.token_count` for the active session.
- [ ] If `token_count >= TOKEN_COMPACTION_THRESHOLD` (default 50,000 — configurable):
  - Triggers `SummarizationService.summarize(sessionId)` synchronously (awaited before the chat response).
  - Emits `{ type: 'context_compacted' }` WS event.
  - Continues with the user's request using the compacted context.
- [ ] Token count updated after each AI response: add response tokens to `chat_sessions.token_count`.
- [ ] Compaction threshold configurable via workspace settings (WORKSPACE_OWNER+).
- [ ] Compaction does NOT happen in the middle of a workflow step — only between steps.

## Tests to Write
- Token count below threshold → no compaction.
- Token count at threshold → compaction triggered before response.
- Compaction during workflow step → deferred to next step boundary.
- Token count updated correctly after response.

## Definition of Done
- Auto-compaction transparent to user. Context preserved post-compaction. Tests pass.
