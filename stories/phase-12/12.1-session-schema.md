# Story 12.1 — Session Management Database Schema

## Phase

Phase 12: Session Isolation & Memory Lifecycle

## Dependencies

- Story 1.2 must be `DONE`.
- Story 6.7 must be `DONE` (owns `chat_messages` table).

## Status

`[ ] PENDING`

## Description

Create the `chat_sessions` and `chat_messages` tables with strict foreign keys ensuring messages cannot bleed across Jira tickets/sessions.

## Acceptance Criteria

- [ ] `chat_sessions` table:

  | Column        | Type                                     |
  | ------------- | ---------------------------------------- |
  | id            | UUID (v7) PK                             |
  | workspace_id  | UUID FK → workspaces.id                  |
  | user_id       | UUID FK → users.id                       |
  | jira_story_id | VARCHAR(100) NULL                        |
  | status        | ENUM ('ACTIVE', 'COMPACTED', 'ARCHIVED') |
  | token_count   | INTEGER DEFAULT 0                        |
  | created_at    | TIMESTAMPTZ DEFAULT NOW()                |
  | updated_at    | TIMESTAMPTZ DEFAULT NOW()                |

- [ ] `chat_messages` table:

  | Column      | Type                                         |
  | ----------- | -------------------------------------------- |
  | id          | UUID (v7) PK                                 |
  | session_id  | UUID FK → chat_sessions.id ON DELETE CASCADE |
  | role        | ENUM ('user', 'assistant', 'system', 'tool') |
  | content     | TEXT NOT NULL                                |
  | agent_name  | VARCHAR(100) NULL                            |
  | token_count | INTEGER NULL                                 |
  | created_at  | TIMESTAMPTZ DEFAULT NOW()                    |

- [ ] `session_id` FK with `ON DELETE CASCADE` ensures messages are deleted when session is purged.
- [ ] Unique constraint on `(workspace_id, user_id, jira_story_id)` where jira_story_id IS NOT NULL — prevents duplicate active sessions per ticket.
- [ ] `SessionRepository`: `create()`, `getById()`, `getActiveByStory()`, `deleteMessages()`, `updateTokenCount()`.
- [ ] **`active_sessions` table** (canonical definition here): `id UUID PK DEFAULT gen_random_uuid(), workspace_id UUID FK, user_id UUID FK, jira_story_id TEXT, context_snapshot JSONB, status TEXT CHECK IN ('active','suspended','completed'), created_at TIMESTAMPTZ DEFAULT NOW(), updated_at TIMESTAMPTZ DEFAULT NOW()`.
- [ ] **`chat_messages` — DO NOT redefine.** The table is created in Story 6.7. This story adds two columns via ALTER TABLE migration: `ALTER TABLE chat_messages ADD COLUMN session_id UUID REFERENCES sessions(id)`, `ALTER TABLE chat_messages ADD COLUMN token_count INTEGER DEFAULT 0`.
