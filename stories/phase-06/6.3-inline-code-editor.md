# Story 6.3 — Center-Stage Inline Code Editor

## Phase

Phase 6: Visual Workspace IDE & Live Code Editor

## Dependencies

- Story 6.1, 6.2 must be `DONE`.
- Story 6.6 should be considered a soft dependency for the read-only mode feature — `execution_mode` is stored in workspace settings (Story 6.6). If 6.6 is not done, default `execution_mode` to `'MANUAL'` (editor always writable).

## Status

`[ ] PENDING`

## Description

Integrate Monaco Editor as the center-panel code editor. Stream AI file updates live into the editor via WebSockets without disrupting the user's scroll position or cursor.

## Acceptance Criteria

- [ ] `GET /api/workspaces/:id/files/content?path=...` — returns file content as text.
- [ ] Monaco Editor mounted in the Center Panel, loading the requested file.
- [ ] Language auto-detected from file extension and applied to Monaco model.
- [ ] WebSocket event `{ type: 'file_update', payload: { path, content } }` applies the new content to the Monaco model using `editor.getModel().setValue()` — but only if the user is NOT currently editing (cursor is not active).
- [ ] If the user IS editing when a WS update arrives: show a banner "AI has updated this file — click to apply" instead of overwriting.
- [ ] Tab bar above the editor showing open files; tab closes on X click.
- [ ] Unsaved indicator (dot) on tabs with local edits.
- [ ] Read-only mode when `execution_mode === 'AUTO'` and a workflow is RUNNING.
- [ ] Keyboard shortcut `Ctrl+S` / `Cmd+S` triggers save (story 6.4).

## Technical Notes

- `Ctrl+S` triggers the save handler from Story 6.4. Add an event listener in the Monaco editor's `onMount` callback but stub the save function (no-op) until Story 6.4 is implemented.
