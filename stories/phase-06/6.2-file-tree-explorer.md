# Story 6.2 — Remote File Tree Explorer Component

## Phase

Phase 6: Visual Workspace IDE & Live Code Editor

## Dependencies

- Story 6.1, 3.2 must be `DONE`.

## Status

`[ ] PENDING`

## Description

Build a directory tree browser in the Left Sidebar that queries the backend filesystem and highlights files with uncommitted AI modifications in yellow.

## Acceptance Criteria

- [ ] `GET /api/workspaces/:id/files?path=/` — returns directory listing: `[{ name, type: 'file'|'dir', path, hasUnstagedChanges: boolean }]`.
- [ ] `hasUnstagedChanges` determined by running `git status --porcelain` on the workspace repo.
- [ ] Directory tree lazily loads children on expand (click to expand directory).
- [ ] Files with `hasUnstagedChanges: true` displayed with a yellow dot indicator.
- [ ] Clicking a file loads it in the Center Editor (story 6.3).
- [ ] Collapsible directory nodes with chevron icons.
- [ ] Search/filter input at top of file tree.
- [ ] Repo selector dropdown if multiple repos are connected.
- [ ] Right-click context menu: "Reveal in Explorer", "Copy Path".
- [ ] File icons based on extension (using `lucide-react` icons — see Technical Notes).

## Technical Notes

- Backend: use `fs.promises.readdir` with `{ withFileTypes: true }`.
- Path security: sanitize input path, reject traversal outside workspace root.
- Git status: `simple-git().status()` to get modified file list.
- "Reveal in Explorer" context menu item copies the server-side file path to the clipboard (uses `navigator.clipboard.writeText(filePath)`). It does NOT attempt OS-level reveal — that is not possible in a web browser.
- File type icons: use `lucide-react` icons (already installed) mapped by extension — `FileText` for text/code, `FileCode` for .ts/.js/.py, `Folder`/`FolderOpen` for directories. Do NOT add `react-file-icon` as a dependency.
