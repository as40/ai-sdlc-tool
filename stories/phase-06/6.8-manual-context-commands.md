# Story 6.8 — Manual Context Commands

## Phase

Phase 6: Visual Workspace IDE & Live Code Editor

## Dependencies

- Story 6.7 must be `DONE`.

## Status

`[ ] PENDING`

## Description

Implement `/clear` and `/compact` slash commands in the chat interface for explicit session context management.

## Acceptance Criteria

- [ ] `/clear` command:
  - Intercepted client-side before sending to server.
  - Calls `DELETE /api/sessions/:sessionId/messages`.
  - Backend truncates `chat_messages` for the session (hard delete).
  - UI clears the chat history instantly.
  - System chip displayed: "Chat cleared".
- [ ] `/compact` command:
  - Calls `POST /api/sessions/:sessionId/compact`.
  - Backend calls summarization pipeline (story 12.5 — can be a stub now that just returns a placeholder summary).
  - Chat history replaced with a single system message containing the summary.
  - System chip displayed: "Context Compacted — X messages summarized".
- [ ] Both commands recognized by the slash command autocomplete dropdown (story 6.7).
- [ ] `/clear` requires confirmation dialog: "This will erase all chat history for this session. Continue?".
- [ ] '/compact' command: if Story 12.5 is not yet implemented, return `{ type: 'system', content: 'Context compaction not yet available.' }` as a WS message. Do not leave the call hanging or return 501.
- [ ] `DELETE /api/sessions/:sessionId/messages` and `POST /api/sessions/:sessionId/compact` require `requireAuth` + `requireRole('DEVELOPER', 'WORKSPACE_OWNER', 'SUPER_ADMIN')`. `VIEWER` must receive 403.
