# Story 6.7 — Hybrid Chat Interface & Slash Commands

## Phase
Phase 6: Visual Workspace IDE & Live Code Editor

## Dependencies
- Story 5.4, 6.1 must be `DONE`.

## Status
`[ ] PENDING`

## Description
Build the chat input window where natural language prompts are sent to the orchestrator, and implement slash command autocomplete to target specific agents directly.

## Acceptance Criteria
- [ ] Chat panel in the Center Area (tab: "Chat" vs "Editor").
- [ ] Chat history renders messages: user messages (right-aligned), agent responses (left-aligned with agent name badge).
- [ ] Message input at bottom with send button and Shift+Enter for newlines.
- [ ] `POST /api/workspaces/:id/chat` — sends a message.
  - Request: `{ sessionId: string; message: string; agentTarget?: string }`.
  - Routes to Orchestrator Supervisor by default, or to named agent if `agentTarget` is provided.
  - Streams response tokens via WebSocket: `{ type: 'chat_token', payload: { token, done: boolean } }`.
- [ ] Typing `/` triggers an autocomplete dropdown showing all registered agent names from the registry.
- [ ] Selecting an agent from autocomplete sets `agentTarget` and prefixes message with `@agentName: `.
- [ ] Streaming tokens displayed in real-time in the chat bubble as they arrive.
- [ ] "Stop" button cancels the current AI stream (sends cancel signal to backend).
- [ ] `chat_messages` table: `id, session_id, role ('user'|'assistant'|'system'), content, agent_name, created_at`.
