## Description

Add Save and Discard actions for human manual edits. Save triggers server disk write, hash recalculation, and incremental pgvector sync for the modified file.

## Acceptance Criteria

- [ ] `PUT /api/workspaces/:id/files/content` — saves file content to disk.
  - Request: `{ path: string; content: string }`.
  - Validates path is within workspace root.
  - Writes content to disk using `fs.promises.writeFile`.
  - Recalculates SHA-256 hash of the new content.
  - Triggers incremental re-vectorization for the single file (calls EmbeddingWorker for just that file).
  - Returns `{ hash: string; updatedAt: string }`.
- [ ] `DELETE /api/workspaces/:id/files/content?path=...` — discards changes by running `git checkout -- <path>`.
  - Returns the reverted file content.
- [ ] Frontend "Save" button (+ Ctrl+S): calls PUT endpoint, clears unsaved indicator.
- [ ] Frontend "Discard" button: shows confirmation dialog, calls DELETE endpoint, reloads file content in editor.
- [ ] Both actions emit a WebSocket event to sync other connected clients: `{ type: 'file_saved', payload: { path, hash } }`.
- [ ] 'Discard' confirmation uses shadcn `AlertDialog` component (install via `npx shadcn-ui@latest add alert-dialog` or create manually in `src/components/ui/alert-dialog.tsx`). Dialog title: 'Discard changes?', description: 'This will permanently discard all unsaved changes to this file.', confirm button variant: `destructive`.
