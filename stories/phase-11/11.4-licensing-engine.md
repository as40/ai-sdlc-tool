# Story 11.4 — Call-Home Licensing Engine

## Phase

Phase 11: IP Protection & Enterprise Distribution

## Dependencies

- Story 11.3 must be `DONE`.

## Status

`[ ] PENDING`

## Description

Build an external lightweight licensing microservice and implement license validation in the SDLC engine that pings it on startup and every 24 hours, shutting down if the license is revoked.

## Acceptance Criteria

- [ ] **Licensing Microservice** (separate minimal Express app):
  - `POST /api/licenses/validate` — accepts `{ licenseKey: string; instanceId: string }`.
  - Returns `{ valid: boolean; expiresAt: string; features: string[] }`.
  - License records stored in a separate lightweight database.
  - Hosted independently from the main application.
- [ ] **SDLC Engine License Client**:
  - `LicenseService` calls the licensing microservice on startup.
  - If `valid: false` or network unreachable after 3 retries: log error and `process.exit(1)`.
  - Successful validation: cache the result in memory with `expiresAt`.
  - Cron job every 24 hours re-validates. If validation fails: graceful shutdown (complete active workflows, then exit).
  - `LICENSE_KEY` and `LICENSE_SERVER_URL` from env vars.
- [ ] License check bypassed when `NODE_ENV=development` and `SKIP_LICENSE_CHECK=true`.
- [ ] Add `packages/license-server` to `pnpm-workspace.yaml` packages array. Add `packages/license-server/package.json` with name `@ai-sdlc/license-server`.
- [ ] On license expiry, complete all active workflows (wait up to 60 seconds) before shutting down. After 60 seconds, force-exit regardless of workflow state.

## Tests to Write

- Valid license → server starts.
- Invalid license → server exits with code 1.
- Network error after 3 retries → server exits.
- Development + SKIP_LICENSE_CHECK → bypassed.
- 24-hour re-validation → calls service again.

## Definition of Done

- License validation works. Revoked license shuts down server. Dev bypass works. Tests pass.
