# Story 11.3 — Distroless Dockerization

## Phase
Phase 11: IP Protection & Enterprise Distribution

## Dependencies
- Story 11.1, 11.2 must be `DONE`.

## Status
`[ ] PENDING`

## Description
Refactor the backend Dockerfile into a multi-stage build that produces a minimal distroless image containing no source files, no markdown prompts, and a minimal attack surface.

## Acceptance Criteria
- [ ] Multi-stage Dockerfile:
  - **Stage 1 (build)**: `node:20-slim` — install dependencies, compile TypeScript, encrypt assets, build binary.
  - **Stage 2 (runtime)**: `gcr.io/distroless/nodejs20-debian12` — copy only the compiled binary and encrypted assets.
- [ ] Final image contains:
  - The compiled server binary.
  - Encrypted `.enc` asset files.
  - NO `.js`, `.ts`, `.md`, `.json` source files.
  - NO `node_modules` directory.
  - NO shell or package manager.
- [ ] Image built via: `docker build -t ai-sdlc-backend .`.
- [ ] `docker run --env-file .env ai-sdlc-backend` starts the server successfully.
- [ ] Image size documented and must be < 200MB.
- [ ] `docker scan` or `trivy` shows no HIGH or CRITICAL CVEs in the final image.

## Files to Create
```
packages/backend/
├── Dockerfile
└── .dockerignore
```

## Tests to Write
- Verify final image has no `.js` files: `docker run --entrypoint sh image find / -name "*.js"` → empty (will fail in distroless — use a build stage check instead).
- Container starts and health check passes.

## Definition of Done
- Image builds. Server runs in container. No source files in final image. No HIGH CVEs.
