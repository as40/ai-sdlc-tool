# Story 11.1 — Backend Binary Compilation

## Phase
Phase 11: IP Protection & Enterprise Distribution

## Dependencies
- All Phase 1-10 stories should be `DONE` before Phase 11.

## Status
`[ ] PENDING`

## Description
Implement a build step using `bun` to compile the entire Express backend and all its dependencies into a single standalone executable file for IP-protected distribution.

## Acceptance Criteria
- [ ] `pnpm --filter backend build:binary` produces a single executable at `dist/ai-sdlc-server` (or `.exe` on Windows).
- [ ] Tool: `bun build --compile` (preferred) or `pkg` as fallback.
- [ ] Executable includes all runtime dependencies bundled.
- [ ] Executable accepts environment variables from `.env` or system env (same as dev server).
- [ ] `GET /health` still returns 200 from the compiled binary.
- [ ] Binary size documented in build output.
- [ ] CI pipeline step: `build:binary` runs and artifact is uploadable.
- [ ] Build script excludes: `.env` files, test files (`*.test.ts`), dev dependencies.

## Files to Create
```
packages/backend/
├── scripts/
│   └── build-binary.sh
└── package.json           (add build:binary script)
```

## Tests to Write
- Build produces a file at expected path.
- Binary executes and health check returns 200 (integration test).

## Definition of Done
- Binary compiles without errors. Health check passes from binary. CI artifact uploaded.
