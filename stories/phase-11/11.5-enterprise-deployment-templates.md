# Story 11.5 — Enterprise Deployment Templates

## Phase

Phase 11: IP Protection & Enterprise Distribution

## Dependencies

- Story 11.3 must be `DONE`.

## Status

`[ ] PENDING`

## Description

Create Kubernetes Helm Charts and docker-compose templates for enterprise clients to deploy the full application stack within their own VPCs.

## Acceptance Criteria

- [ ] **docker-compose template** (`deploy/docker-compose.production.yml`):
  - Services: `backend`, `frontend`, `postgres`, `pgvector`, `redis`.
  - Uses image references (not build contexts) — client pulls from a private registry.
  - Environment variable stubs documented via `.env.production.example`.
  - Health checks defined for all services.
  - Networks: isolated internal network + frontend-facing network.
- [ ] **Kubernetes Helm Chart** (`deploy/helm/ai-sdlc-tool/`):
  - `Chart.yaml` with correct metadata.
  - `values.yaml` with all configurable values documented.
  - Templates: `deployment.yaml`, `service.yaml`, `ingress.yaml`, `configmap.yaml`, `secret.yaml`, `persistentvolumeclaim.yaml` (for workspace storage).
  - HorizontalPodAutoscaler template.
  - `NOTES.txt` with post-install instructions.
- [ ] Helm chart passes `helm lint` with no errors.
- [ ] `helm template` renders valid Kubernetes manifests.
- [ ] README in `deploy/` explaining both deployment options.

## Technical Notes

- Redis setup for production session store: add `redis` and `connect-redis` packages to backend. Configure in `src/middleware/session.middleware.ts` to use `RedisStore` when `NODE_ENV === 'production'` and `REDIS_URL` env var is set. Add `REDIS_URL=redis://localhost:6379` to `.env.example`.
- The `deploy/README.md` referenced in this story is exempt from the CLAUDE.md 'no README updates' rule because it is a deployment operations document, not the root project README.
