# Story 11.2 — Prompt & Asset Encryption

## Phase

Phase 11: IP Protection & Enterprise Distribution

## Dependencies

- Story 1.3 (CryptoService), 11.1 must be `DONE`.

## Status

`[ ] PENDING`

## Description

Write a pre-build script that encrypts the /agents, /workflows, and /plugins directories. Update the Express engine to decrypt these assets in RAM on application boot.

## Acceptance Criteria

- [ ] `scripts/encrypt-assets.ts` — pre-build script:
  - Reads all `.md` and `.json` files in `src/agents/`, `src/workflows/`, `src/plugins/`.
  - Encrypts each file's content with AES-256-GCM (using `CryptoService`).
  - Writes encrypted versions to `dist/assets/` with `.enc` extension.
  - Original source files are NOT included in the final binary/Docker image.
- [ ] `AssetLoader` service runs at app startup:
  - Reads encrypted `.enc` files from `dist/assets/`.
  - Decrypts them in memory using `ASSET_ENCRYPTION_KEY` env var.
  - Serves decrypted content to `RegistryLoader` (story 5.1) — never writes to disk.
- [ ] `RegistryLoader` updated to accept content from `AssetLoader` instead of reading files directly.
- [ ] `ASSET_ENCRYPTION_KEY` is different from `ENCRYPTION_KEY` (separate key per concern).
- [ ] Build output directory contains no `.md` or `.js` source prompt files.
- [ ] Add `ASSET_ENCRYPTION_KEY=<32-byte-hex>` to `packages/backend/.env.example` with a comment: `# 32-byte hex key for encrypting bundled prompt assets — generate with: node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"`.​

## Files to Create

```
packages/backend/
├── scripts/encrypt-assets.ts
└── src/
    └── registry/
        └── asset-loader.service.ts
```

## Tests to Write

- `encrypt-assets` script produces `.enc` files with no original files present.
- `AssetLoader` decrypts file contents matching originals.
- Wrong decryption key → startup fails with clear error.

## Definition of Done

- Dist directory contains no readable source prompts. Agents/skills still function. Tests pass.
