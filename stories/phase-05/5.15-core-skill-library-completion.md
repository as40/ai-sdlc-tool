# Story 5.15 — Core Skill Library Completion

## Phase

Phase 5: Core AI Orchestration Engine

## Dependencies

- Story 5.3 must be `DONE` (SkillRouter + built-in file/terminal/search skills).
- Story 3.1 should be `DONE` (git provider auth — git skills operate on the cloned repo).

## Status

`[ ] PENDING`

## Description

Story 5.3 authors the file, terminal, search, and memory skills. But the agents in the persona library (5.13) reference git, test, and Jira skills that no story creates. This story completes the skill catalog so every skill an agent's allow-list names actually exists, exposed through the same `SkillRouter` contract (provider-neutral tool definitions, Zod-validated input, structured errors, output truncation).

## Skills to Author (7)

| Skill        | Input                                 | Behavior                                                                                                                                                                                                    |
| ------------ | ------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `git_status` | `{}`                                  | Returns porcelain status of the workspace repo (via `simple-git`).                                                                                                                                          |
| `git_diff`   | `{ staged?: boolean; path?: string }` | Returns the diff, truncated via the 5.3 truncation util.                                                                                                                                                    |
| `git_branch` | `{ name: string; from?: string }`     | Creates/switches a branch. Name sanitized; no shell interpolation.                                                                                                                                          |
| `git_commit` | `{ message: string }`                 | Stages and commits. **HITL-gated** (write action). Never pushes.                                                                                                                                            |
| `run_tests`  | `{ scope?: string }`                  | Runs the workspace's configured test command (from the verification profile, Story 5.10) in the sandbox; distinct from raw `run_terminal_command` so it can be allow-listed narrowly and its output parsed. |
| `run_linter` | `{ path?: string; fix?: boolean }`    | Runs the configured lint command; returns structured findings.                                                                                                                                              |
| `jira_read`  | `{ issueKey: string }`                | Reads a Jira issue's fields via the stored Jira config (Story 7.1). Read-only.                                                                                                                              |

## Acceptance Criteria

- [ ] Each skill has a `skill.config.json` (name, description, JSON-Schema `input_schema`, handler ref) and a handler implementing the skill; all are auto-loaded by the registry (5.1) and exposed by the SkillRouter (5.3).
- [ ] All skills validate input with Zod and return structured errors (`{ error, code }`) on failure — reusing the 5.3 error contract.
- [ ] Git skills operate **only** within the workspace repo directory; paths are sanitized and validated against `WORKSPACE_BASE_PATH/{workspaceId}` (same guard as `read_file`/`write_file`). `git_commit` never runs `git push` (push is Phase 8's PR automation, human-gated).
- [ ] `git_commit`, `git_branch`, `run_linter --fix` are treated as mutating actions and respect the agent's `hitl_required_for` config.
- [ ] `run_tests` and `run_linter` execute in the sandbox context when available (Story 9); outside a sandbox they return a structured `SANDBOX_REQUIRED` error rather than running on the host.
- [ ] Output of `git_diff`, `run_tests`, `run_linter` passes through `truncateToolOutput` (5.3) so large output cannot blow the context window.
- [ ] `jira_read` returns a compact projection (summary, description, acceptance criteria, status) — not the raw Jira payload — to keep tokens low.
- [ ] Every skill in the 5.13 agent allow-lists now resolves to a real registered skill (integration check).
