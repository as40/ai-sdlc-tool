# Story 5.13 — Core Agent Persona Library

## Phase

Phase 5: Core AI Orchestration Engine

## Dependencies

- Story 5.1 must be `DONE` (registry loader + agent frontmatter schema incl. `model_tier`).
- Story 5.3 must be `DONE` (skills the agents reference must exist to allow-list them).

## Status

`[ ] PENDING`

## Description

Author the complete set of production agent personas the orchestration engine routes between. Story 5.1 ships only an _example_ `developer-agent.md`; the supervisor, reviewer, and devops agents are defined ad hoc in other stories. This story consolidates and authors the full, first-class agent library so the AI can cover the SDLC. Each agent is a single `.md` file with YAML frontmatter (loaded by the registry) plus a carefully written persona body — this content is the largest lever on output quality after the base prompt.

## Acceptance Criteria

- [ ] Each agent `.md` exists in `src/agents/` with valid frontmatter: `name`, `description`, `version`, `model_tier`, `skills` (allow-listed skill names), `hitl_required_for` (array of action categories).
- [ ] The registry loader (5.1) loads all 8 without error; `GET /api/registry/agents` returns them.
- [ ] Each persona body follows a consistent template: **Role**, **Operating principles** (references the workspace rules/standards, not a copy of them), **What you must do / must never do**, **Output expectations** (e.g. "return unified diffs, not full-file rewrites" — aligns with Story 12.8), **When to ask for human input**.
- [ ] Each agent's `skills` allow-list contains only skills that exist in the `SkillRegistry`; loader logs a warning and drops any unknown skill reference (does not crash).
- [ ] `orchestrator-supervisor.md`, `reviewer.md`, and `devops.md` supersede the ad-hoc definitions referenced in Stories 5.4, 5.12, and 9.1 — those stories consume these canonical files rather than defining their own. (Note this explicitly to avoid duplicate personas.)
- [ ] The `developer.md` persona replaces the "example" `developer-agent.md` from Story 5.1 as the canonical production version.
- [ ] Personas are stack-agnostic — stack-specific guidance comes from the standards library (Story 5.16) merged by the prompt compiler (5.2), NOT hardcoded in the persona. Keep personas DRY.
