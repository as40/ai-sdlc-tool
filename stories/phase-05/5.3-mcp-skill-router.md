# Story 5.3 — Model Context Protocol (MCP) Skill Router

## Phase

Phase 5: Core AI Orchestration Engine

## Dependencies

- Story 5.1 must be `DONE`.

## Status

`[ ] PENDING`

## Description

Implement the MCP skill router that exposes backend skill functions as standard tool definitions to the AI, and executes them when the AI issues a tool call.

## Acceptance Criteria

- [ ] `SkillRouter` converts each `SkillDefinition` into a **provider-neutral** tool definition object: `{ name, description, input_schema }` (JSON Schema). The `LLMGateway` provider adapters (Story 5.4) are responsible for translating this format to the provider's wire format (Anthropic `tools[]`, OpenAI `functions[]`, etc.). `SkillRouter` itself never references a specific provider format.
- [ ] `SkillRouter.execute(skillName: string, input: unknown): Promise<unknown>` — validates input against the skill's JSON schema with Zod, then calls the skill's handler function.
- [ ] Built-in skills implemented:
  - `read_file(path)` — reads a file within the workspace directory. Rejects paths outside workspace root (path traversal prevention).
  - `write_file(path, content)` — writes content to a file. Validates path is within workspace root.
  - `list_directory(path)` — lists files in a directory.
  - `run_terminal_command(command)` — executes a shell command inside the sandbox container via Docker exec. NOT available outside sandbox context.
  - `search_codebase(query)` — performs a similarity search on `code_embeddings` using the `EmbeddingRepository`.
  - `save_memory(type, key, content)` — saves a fact to `workspace_memory` via `MemoryRepository.upsert()`. Valid types: `pattern`, `constraint`, `architectural_decision`. (`preference` and `checkpoint` types are written by system services, not by agents directly — reject those with `INVALID_MEMORY_TYPE`.) Requires Story 4.4 to be `DONE`; stub returns `{ saved: false, reason: 'memory_store_not_available' }` until then.
  - `recall_memory(type?, query?)` — retrieves memories from `workspace_memory`. Optional filters: `type` to narrow by memory type, `query` for keyword match against `key` and `content`. Returns an array of `{ key, content, confidence }` objects. Capped at 10 results. Requires Story 4.4.
- [ ] **Tool output truncation** — all skill results pass through `truncateToolOutput(content, maxTokens)` before being returned to the AI:
  - `read_file`: truncates at `MAX_TOOL_OUTPUT_TOKENS` (default 4 000, configurable via env var). If truncated, appends: `[...truncated at {N} tokens. Use search_codebase(query) to locate specific sections.]`
  - `list_directory`: caps at 200 entries. If more exist, appends: `[...{N} more entries omitted. Narrow the path to see more.]`
  - `search_codebase`: returns top 5 results maximum (already limited by similarity search `LIMIT`; enforce here too as a safety cap).
  - `run_terminal_command`: stdout/stderr combined output truncated at `MAX_TOOL_OUTPUT_TOKENS`. Truncation suffix: `[...command output truncated. Re-run with a narrower command to see more.]`
  - `write_file`: no truncation — returns a short confirmation string, not file content.
  - `truncateToolOutput` is a pure utility function — separately testable.
- [ ] All skill inputs validated with Zod before execution.
- [ ] Skill execution errors return a structured error to the AI: `{ error: string; code: string }`.
- [ ] Skill execution results and errors are logged to the `workflow_executions` audit trail.
- [ ] `GET /api/registry/skills/tool-definitions` — returns the array of provider-neutral tool definitions (for debugging).

## Security Notes

- `read_file` and `write_file` MUST validate that the resolved path starts with `WORKSPACE_BASE_PATH/{workspaceId}`. Reject anything else with `SECURITY_VIOLATION` error code.
- `run_terminal_command` is ONLY callable when a sandbox container is running for the workspace. Otherwise returns error.

## Tests to Write

- `read_file` with valid path → returns content.
- `read_file` with path traversal (`../../etc/passwd`) → SECURITY_VIOLATION error.
- `write_file` outside workspace root → SECURITY_VIOLATION.
- `search_codebase` → calls EmbeddingRepository.similaritySearch.
- `run_terminal_command` without sandbox → returns error.
- Invalid skill name → `SKILL_NOT_FOUND` error.
- Input fails Zod schema → `VALIDATION_ERROR`.
- `save_memory` with `type: 'checkpoint'` → `INVALID_MEMORY_TYPE` error.
- `save_memory` with valid type → memory upserted via MemoryRepository.
- `recall_memory` with `type: 'pattern'` → returns only pattern memories, capped at 10.
- `read_file` result exceeds `MAX_TOOL_OUTPUT_TOKENS` → output truncated, suffix appended.
- `read_file` result within limit → output returned in full, no suffix.
- `list_directory` with 250 entries → capped at 200, omission notice appended.
- `run_terminal_command` with large stdout → truncated to `MAX_TOOL_OUTPUT_TOKENS`.
- `truncateToolOutput` with content under limit → returns content unchanged.
- Tool definitions returned by `SkillRouter` use provider-neutral JSON Schema format, not Anthropic-specific format.

## Definition of Done

- All tests pass. Path traversal tests definitely pass. Tool definitions are provider-neutral. All content-returning skills truncate oversized output before returning it to the AI.
