# Story 5.1 — Dynamic Registry Loader

## Phase

Phase 5: Core AI Orchestration Engine

## Dependencies

- Story 1.1 (engine directories) must be `DONE`.

## Status

`[ ] PENDING`

## Description

Write an initialization script that scans the `/agents`, `/skills`, and `/plugins` directories on startup, reads their configuration metadata files, and loads them into an in-memory registry for use by the orchestration engine.

## Acceptance Criteria

- [ ] `RegistryLoader` service scans the three directories on app startup.
- [ ] Each agent is defined by an `.md` file with YAML frontmatter:
  ```yaml
  ---
  name: developer-agent
  description: Writes and modifies source code
  version: 1.0.0
  ---
  [System prompt markdown...]
  ```
- [ ] Each skill is defined by a `skill.config.json`:
  ```json
  {
    "name": "read_file",
    "description": "...",
    "version": "1.0.0",
    "handler": "./read-file.skill.ts"
  }
  ```
- [ ] Each plugin folder `plugins/{tech-stack}/` contains a `standards.md` file.
- [ ] In-memory registry:
  - `AgentRegistry`: `Map<string, AgentDefinition>` — keyed by agent name.
  - `SkillRegistry`: `Map<string, SkillDefinition>` — keyed by skill name.
  - `PluginRegistry`: `Map<string, PluginDefinition>` — keyed by tech-stack name.
- [ ] `GET /api/registry/agents` — lists all loaded agents (admin only). Requires `requireAuth` + `requireRole('WORKSPACE_OWNER', 'SUPER_ADMIN')`. A `DEVELOPER` or `VIEWER` must not be able to read the agent/skill registry directly.
- [ ] `GET /api/registry/skills` — lists all loaded skills (admin only). Requires `requireAuth` + `requireRole('WORKSPACE_OWNER', 'SUPER_ADMIN')`. A `DEVELOPER` or `VIEWER` must not be able to read the agent/skill registry directly.
- [ ] Registry is a singleton; hot-reloading not required (restart to reload).
- [ ] If a config file is malformed (invalid YAML/JSON), log a warning and skip it — do NOT crash the app.

## Technical Notes

- Use `gray-matter` for YAML frontmatter parsing.
- `fs.readdirSync` on startup (synchronous is fine here — happens before server starts).
- Registry exposed via dependency injection container.
- Role decision rationale: the registry contains internal model routing config that could expose provider selection and cost information. Restrict to owners/admins.

## Tests to Write

- Valid agent .md → loaded into AgentRegistry.
- Valid skill config → loaded into SkillRegistry.
- Malformed YAML frontmatter → skipped, warning logged, other agents still load.
- Missing handler file in skill config → warning logged, skill skipped.
- Registry returns correct counts after loading fixture directory.
