# Story 5.5 — Human-in-the-Loop (HITL) Interceptor & Revisor

## Phase
Phase 5: Core AI Orchestration Engine

## Dependencies
- Story 5.4 must be `DONE`.

## Status
`[ ] PENDING`

## Description
Implement the HITL execution interceptor that halts agent transitions in MANUAL mode and waits for a human approval signal. When revision is requested, compile the feedback + git diff into a targeted revision payload for the agent.

## Acceptance Criteria
- [ ] In `MANUAL` execution mode, after each major agent output the orchestrator transitions to `AWAITING_APPROVAL` and emits a WebSocket event.
- [ ] `POST /api/workspaces/:id/workflows/:workflowId/approve` — approves the current output and resumes the workflow.
  - Requires `DEVELOPER` role minimum.
  - Transitions: `AWAITING_APPROVAL → RUNNING`.
- [ ] `POST /api/workspaces/:id/workflows/:workflowId/revise` — rejects and requests revision.
  - Request: `{ feedback: string }`.
  - Transitions: `AWAITING_APPROVAL → REVISION_REQUESTED`.
  - Builds a revision payload: `{ feedback, gitDiff: string (output of git diff), originalOutput }`.
  - Feeds the revision payload back into the agent's next turn as a user message.
  - Transitions: `REVISION_REQUESTED → RUNNING`.
- [ ] Git diff is obtained by running `git diff` in the workspace repo directory (using `simple-git`).
- [ ] Revision feedback is stored in `workflow_executions` history (JSONB).
- [ ] Maximum revision cycles: 5 (configurable). Exceeding forces COMPLETED with last output.
- [ ] WebSocket events:
  - `{ type: 'awaiting_approval', payload: { workflowId, agentOutput, gitDiff } }`.
  - `{ type: 'revision_started', payload: { workflowId, revisionCount } }`.
