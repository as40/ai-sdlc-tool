# Story 5.2 — System Prompt & Markdown Standards Compiler

## Phase
Phase 5: Core AI Orchestration Engine

## Dependencies
- Story 5.1, 3.3 (tech stack scanner) must be `DONE`.

## Status
`[ ] PENDING`

## Description
Build a compiler utility that merges an agent's baseline persona with framework-specific guidelines from the plugin for the detected tech stack to produce a complete, final system prompt.

## Acceptance Criteria
- [ ] `PromptCompiler` service with method: `compile(agentName: string, techStack: string): string`.
- [ ] Compilation order (sections merged top-to-bottom):
  1. Immutable system header (from `src/standards/system-header.md`)
  2. Agent persona (from `AgentRegistry.get(agentName)`)
  3. Plugin standards (from `PluginRegistry.get(techStack).standards`)
  4. Workspace-level custom override (from `ai_configurations` persona field — story 13)
  5. Immutable system footer (from `src/standards/system-footer.md`)
- [ ] If no plugin exists for the detected tech stack: skip step 3 gracefully.
- [ ] If no workspace override: skip step 4 gracefully.
- [ ] Compiled prompt must not exceed a configurable `MAX_SYSTEM_PROMPT_TOKENS` (default 8000 tokens).
- [ ] If it exceeds the limit: truncate sections 3 and 4 first (preserve header, persona, footer).
- [ ] Output is a pure string — no Markdown rendering needed at this layer.
- [ ] `GET /api/workspaces/:id/prompt-preview?agent=developer&stack=react` — returns the compiled prompt (WORKSPACE_OWNER+ only) for debugging.

## Technical Notes
- Section headers can use `\n\n---\n\n` as separators.
- Token counting via `tiktoken`.
- Compiler is a pure function for testability — dependencies injected.
