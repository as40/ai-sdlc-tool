# Story 8.2 — Manual Branch Switching Dropdown

## Phase
Phase 8: Automated Branching & Version Control Operations

## Dependencies
- Story 8.1 must be `DONE`.

## Status
`[ ] PENDING`

## Description
Implement a branch control dropdown in the IDE navigation panel allowing manual git branch toggling via the UI.

## Acceptance Criteria
- [ ] `GET /api/workspaces/:id/git/branches` — list all local branches. Marks current branch.
- [ ] `POST /api/workspaces/:id/git/branches/checkout` — checkout a branch.
  - Request: `{ branch: string }`.
  - Runs `git checkout <branch>` via simple-git.
  - Returns error if there are unstaged changes (prompt user to save or discard first).
- [ ] TopNavBar branch dropdown shows current branch name with a branch icon.
- [ ] Dropdown lists all local branches; selecting one calls checkout endpoint.
- [ ] If checkout blocked by unstaged changes: show toast "Unsaved changes — please save or discard before switching branches".
- [ ] "Create new branch" option in dropdown opens a modal with a branch name input.


## Tests to Write
- List branches returns all local branches with current marked.
- Checkout with unstaged changes → 409 Conflict.
- Checkout clean → success, current branch updates.

## Definition of Done
- Branch dropdown renders and functions. Clean checkout works. Dirty-state protection works.
