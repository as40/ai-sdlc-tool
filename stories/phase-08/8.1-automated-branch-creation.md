# Story 8.1 — Automated Feature Branch Creation

## Phase
Phase 8: Automated Branching & Version Control Operations

## Dependencies
- Story 7.3 (Jira context attachment) must be `DONE`.

## Status
`[ ] PENDING`

## Description
Ensure that when a Jira story is attached to a session, the backend automatically creates a feature branch in the workspace repo. All AI file modifications remain as unstaged working-directory changes until final approval.

## Acceptance Criteria
- [ ] Hook in `jira-context.service.ts`: when a story is attached, call `GitService.createFeatureBranch(repoPath, jiraStoryKey)`.
- [ ] Branch name pattern: `feature/{JIRA-KEY}` (e.g., `feature/PROJ-42`).
- [ ] If the branch already exists: checkout the existing branch (do not error).
- [ ] If the base branch (`main` or `master`) doesn't exist: log error, abort branch creation, return warning.
- [ ] `POST /api/workspaces/:id/git/branches` — manual branch creation endpoint.
- [ ] All AI-generated file writes (via `write_file` skill) land as **unstaged** changes in the working directory. Never auto-committed.
- [ ] Current branch name shown in TopNavBar (from `git branch --show-current`).
- [ ] WebSocket event: `{ type: 'branch_created', payload: { branch, repoId } }`.
