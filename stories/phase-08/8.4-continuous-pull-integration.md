# Story 8.4 — Post-Approval Continuous Pull Integration

## Phase

Phase 8: Automated Branching & Version Control Operations

## Dependencies

- Story 8.3 must be `DONE`.

## Status

`[ ] PENDING`

## Description

Create a webhook endpoint that monitors PR merge status on the Git provider. Once a PR is merged, automatically checkout the target branch and run git pull locally.

## Acceptance Criteria

- [ ] `POST /api/webhooks/github` — GitHub webhook endpoint.
  - Validates `X-Hub-Signature-256` header against a workspace-level webhook secret.
  - Handles `pull_request` event with action `closed` and `merged: true`.
  - Identifies the workspace from the `repository.full_name` payload.
  - Triggers `GitService.pullMergedBranch(repoPath, targetBranch)`.
- [ ] `GitService.pullMergedBranch`: `git checkout {targetBranch}` then `git pull origin {targetBranch}`.
- [ ] WebSocket event: `{ type: 'pr_merged', payload: { branch, prUrl } }`.
- [ ] UI shows a toast notification: "PR merged — pulled latest {branch}".
- [ ] `webhook_secrets` table: `id, workspace_id, repo_id, secret_encrypted, created_at`.
- [ ] Admin UI to register a webhook secret per repo (WORKSPACE_OWNER+).
- [ ] Bitbucket webhook support: same pattern with `POST /api/webhooks/bitbucket`.
- [ ] Migration: add `webhook_secrets` table — `id UUID PK, workspace_id UUID FK, repo_id UUID FK, secret TEXT NOT NULL, created_at TIMESTAMPTZ`. Add migration file `packages/backend/src/db/migrations/add-webhook-secrets.ts`.
- [ ] Admin UI for registering webhook secrets: a simple form at `/workspaces/:id/settings/webhooks` (standalone route, migrated to Settings tab in Story 13.1). Form fields: repository dropdown, secret input (write-only). Requires `WORKSPACE_OWNER` role.
