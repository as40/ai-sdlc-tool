# Story 14.1 — Canvas Workflow Builder UI

## Phase

Phase 14: Visual Compilation Engine

## Dependencies

- Story 5.4, 13.1 must be `DONE`.

## Status

`[ ] PENDING`

## Description

Build a drag-and-drop canvas interface allowing users to visually link Agent nodes and export the graph as a JSON execution definition.

## Acceptance Criteria

- [ ] Canvas tab in the Workflows settings page.
- [ ] Uses `React Flow` for the node-based canvas.
- [ ] Node types:
  - **Agent Node**: represents an agent (named, colored by type).
  - **Condition Node**: routes flow based on output evaluation.
  - **Start Node**: entry point.
  - **End Node**: terminal state.
- [ ] Drag agents from a sidebar palette onto the canvas.
- [ ] Connect nodes by drawing edges (output → input).
- [ ] Node configuration panel: click a node to set its agent, execution mode, retry policy.
- [ ] "Export" button converts the visual graph to a JSON execution graph:
  ```json
  {
    "nodes": [{ "id": "n1", "type": "agent", "agentName": "developer-agent" }],
    "edges": [{ "source": "n1", "target": "n2" }],
    "startNode": "n1"
  }
  ```
- [ ] "Save" button calls `POST /api/workspaces/:id/workflow-definitions` with the JSON graph.
- [ ] Saved workflows listed and loadable.
- [ ] Loading state while saving workflow graph: Save button shows 'Saving…' and is disabled. Error on save shows `<Alert variant='error'>` with `detail`.

## Technical Notes

- `@xyflow/react` (React Flow) is not in the CLAUDE.md tech stack list. Add as a confirmed dependency: `pnpm --filter frontend add @xyflow/react`.
- **Route conflict resolution**: `POST /api/workspaces/:id/workflows` is used by Story 5.4 for workflow _executions_. This story's endpoint uses `POST /api/workspaces/:id/workflow-definitions` (CRUD for named, reusable workflow graphs). Update all references in this story from `POST /api/workspaces/:id/workflows` to `POST /api/workspaces/:id/workflow-definitions`.
