# Story 14.5 — AI-Assisted Agent Prompt Generator

## Phase
Phase 14: Visual Compilation Engine

## Dependencies
- Story 13.2, 5.4 must be `DONE`.

## Status
`[ ] PENDING`

## Description
Build a frontend text input where users describe a desired Agent persona in natural language. A backend endpoint uses an LLM to expand the description into a comprehensive structured system prompt that pre-fills the Agent Builder form.

## Acceptance Criteria
- [ ] "Generate from description" section above the AgentPersonaForm (story 13.2).
- [ ] Single textarea: "Describe your agent in plain English (e.g., 'A QA engineer specializing in React component testing')."
- [ ] "Generate" button calls `POST /api/workspaces/:id/agents/generate-persona`.
  - Request: `{ description: string (min 20 chars) }`.
  - Backend LLM prompt: expand the description into structured fields: Role, Coding Standards, Definition of Done.
  - Response: `{ role: string; codingStandards: string; definitionOfDone: string }`.
- [ ] Response pre-fills the corresponding fields in the AgentPersonaForm below.
- [ ] User can review and edit before saving.
- [ ] Loading spinner while generating.
- [ ] Character counts update after pre-fill (form re-validates).

- Description < 20 chars → 400.
- LLM failure → 503 with user-friendly message.
- Pre-filled fields satisfy Zod minimum lengths.

## Definition of Done
- Generated prompt pre-fills form. User can edit and save. Tests pass.
