# Story 14.3 — Secure Workspace File Storage

## Phase
Phase 14: Visual Compilation Engine

## Dependencies
- Story 14.2 must be `DONE`.

## Status
`[ ] PENDING`

## Description
Architect an isolated directory structure on the server to securely store generated workspace configurations without cross-client data bleed.

## Acceptance Criteria
- [ ] Storage root: `STORAGE_BASE_PATH/{workspaceId}/` (env var configurable).
- [ ] Sub-directories: `agents/`, `skills/`, `workflows/`, `uploads/`.
- [ ] `WorkspaceStorageService`:
  - `save(workspaceId, type, name, content)` — writes file to isolated path.
  - `read(workspaceId, type, name)` — reads file; rejects if path resolves outside workspace directory.
  - `list(workspaceId, type)` — lists files of a given type.
  - `delete(workspaceId, type, name)` — deletes file.
- [ ] ALL path operations validated to remain within `STORAGE_BASE_PATH/{workspaceId}/`. Path traversal via `name` is rejected.
- [ ] Directory permissions: `755` for directories, `644` for files.
- [ ] `POST /api/workspaces/:id/storage/agents` — save compiled agent to workspace storage.
- [ ] Saved agents/skills loaded by `RegistryLoader` alongside system defaults.

## Tests to Write
- Write and read file → content matches.
- Path traversal in name (`../../etc`) → rejected.
- Cross-workspace read (wrong workspaceId) → rejected.
- Directory listing returns only files for that workspace/type.

## Definition of Done
- Files stored in isolated directories. Path traversal impossible. Loaded by registry. Tests pass.
