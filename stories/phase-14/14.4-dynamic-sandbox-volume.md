# Story 14.4 — Dynamic Sandbox Volume Mounting

## Phase

Phase 14: Visual Compilation Engine

## Dependencies

- Story 9.1, 14.3 must be `DONE`.

## Status

`[ ] PENDING`

## Description

Update the Docker orchestration script to dynamically mount the custom /agents and /skills directories from workspace storage into the active container at runtime.

## Acceptance Criteria

- [ ] `ComposeGeneratorService` (story 9.1) updated to include volume mounts in the generated `docker-compose.yml`:
  ```yaml
  volumes:
    - {STORAGE_BASE_PATH}/{workspaceId}/agents:/app/agents:ro
    - {STORAGE_BASE_PATH}/{workspaceId}/skills:/app/skills:ro
  ```
- [ ] Mounts are **read-only** (`:ro`) — containers cannot write to the agent/skill config.
- [ ] If the workspace has no custom agents/skills, the volume mount is omitted and system defaults are used.
- [ ] `sandbox/start` endpoint (story 9.2) triggers compose regeneration if workspace config has changed since last compose generation.
- [ ] New container environment variable `WORKSPACE_ID` injected for the containerized agent runtime to reference.

## Tests to Write

- Workspace with custom agents → compose file includes correct volume mounts.
- No custom agents → no volume mount in compose.
- Volume mount paths are read-only.
- Volume mount path is correctly constructed from workspace and repo IDs.
- Env vars are correctly injected into the container spec.
