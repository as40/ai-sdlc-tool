# Story 4.3 — Real-Time Mapping Progress Loader Card

## Phase

Phase 4: Full Vectorization & Long-Term Memory Mapping

## Dependencies

- Story 4.2, 1.4 (WebSocket) must be `DONE`.

## Status

`[ ] PENDING`

## Description

Design and build a workspace dashboard card component that displays an active loading state while a repository is being indexed, driven by live WebSocket progress events.

## Acceptance Criteria

- [ ] Dashboard shows a card per connected repository.
- [ ] Card states: `IDLE`, `INDEXING`, `INDEXED`, `ERROR`.
- [ ] While `INDEXING`:
  - An animated progress bar shows `filesProcessed / totalFiles` percentage.
  - A live counter shows `X chunks created`.
  - An elapsed timer shows time since indexing started.
  - File currently being processed shown as truncated path.
- [ ] `INDEXED` state: shows a green checkmark with total chunks and last indexed timestamp.
- [ ] `ERROR` state: shows error message and a "Retry" button that calls `POST /vectorize`.
- [ ] "Re-index" button (for INDEXED repos) triggers re-vectorization.
- [ ] React hook `useVectorizationStatus(repoId)` subscribes to WebSocket events and maintains local state.
- [ ] Card is shown on a Workspace Overview page accessible to all workspace members.
- [ ] `WorkspaceOverviewPage.tsx` is mounted at route `/workspaces/:id/overview` in `App.tsx`. Requires `ProtectedRoute` with minimum `DEVELOPER` role. This is the first page a user lands on after workspace setup (cloning + env vars done).
- [ ] Navigation flow: after env var setup (Story 3.4), redirect to `/workspaces/:id/overview`. The overview page shows the vectorization progress card and repo cards from Story 3.3.

## Technical Notes

- WebSocket event: `{ type: 'vectorization_progress', payload: { repoId, filesProcessed, totalFiles, chunksCreated, currentFile, status } }`.
- Use React Query to poll `/vectorize/status` as a fallback if WebSocket disconnects.
- Card component must not block the rest of the dashboard while indexing.
- The `useVectorizationStatus` hook uses HTTP polling (`GET /api/workspaces/:id/vectorization/status` every 3 seconds) until Story 6.1 establishes the WebSocket connection. When 6.1 is done, the hook switches to listening for `vectorization_progress` WS events instead of polling. The component interface does not change.
