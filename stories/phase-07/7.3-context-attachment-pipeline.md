# Story 7.3 — Interactive Context Attachment Pipeline

## Phase

Phase 7: Jira Integration & Context Binding

## Dependencies

- Story 7.2, 6.7 must be `DONE`.

## Status

`[ ] PENDING`

## Description

Allow users to click any Jira story to bind it into the active chat session, injecting the full story context into the AI's context window.

## Acceptance Criteria

- [ ] Clicking a Jira story in the sidebar calls `POST /api/sessions/:sessionId/attach-jira`.
  - Request: `{ jiraStoryId: string }`.
  - Fetches the full story detail from Jira: title, description, acceptance criteria, attachments.
  - Creates a `system` message in `chat_messages` with the formatted story context.
  - Updates `workflow_executions.jira_story_id` for the active workflow.
  - Triggers feature branch creation (story 8.1 hook point).
- [ ] Chat panel shows a Jira story "context chip" above the input: `[PROJ-42] Story Title × dismiss`.
- [ ] Dismissing the chip removes the Jira context from the session.
- [ ] `GET /api/sessions/:sessionId/context` — returns attached Jira story if any.
- [ ] Only one Jira story can be attached at a time per session (replacing triggers story 12.2 purge flow).
- [ ] `POST /api/sessions/:sessionId/attach-jira` requires `requireAuth` + `requireRole('DEVELOPER', 'WORKSPACE_OWNER', 'SUPER_ADMIN')`.

## Technical Notes

- Story 8.1 hook point: `POST /api/sessions/:sessionId/attach-jira` emits a `jira_story_attached` internal event. Story 8.1 listens for this event and triggers branch creation. For Phase 7 (before 8.1 is done), the event is emitted but no listener exists — this is expected and non-breaking.
