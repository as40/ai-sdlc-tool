# Story 7.2 — Compact Epic-to-Story Directory

## Phase

Phase 7: Jira Integration & Context Binding

## Dependencies

- Story 7.1, 6.2 must be `DONE`.

## Status

`[ ] PENDING`

## Description

Render a hierarchical list view in the Left Sidebar showing Jira epics and their child stories for the configured projects.

## Acceptance Criteria

- [ ] `GET /api/workspaces/:id/jira/issues` — fetches issues from Jira API.
  - Fetches Epics and Stories for all configured project keys.
  - Groups Stories under their parent Epic.
  - Returns: `[{ epic: { id, key, summary }, stories: [{ id, key, summary, status, assignee }] }]`.
- [ ] Left Sidebar has a "Jira" tab (alongside "Files" tab).
- [ ] Jira tab renders epics as collapsible sections with child stories.
- [ ] Story status shown as colored badge (To Do / In Progress / Done).
- [ ] Clicking a story activates it as the current context (story 7.3).
- [ ] Refresh button to re-fetch from Jira.
- [ ] Loading skeleton while fetching.
- [ ] Search/filter field within the Jira panel.
- [ ] Loading state while fetching Jira epics/stories: show skeleton rows using CSS `animate-pulse rounded bg-zinc-800 h-4` divs (shadcn Skeleton pattern) — no external Skeleton component needed.
- [ ] Jira API errors: backend returns RFC 7807 `{ status: 502, title: 'Jira API Error', detail: '<message from Jira>' }`. Frontend shows `<Alert variant='error'>` with the `detail` and a 'Retry' button.
