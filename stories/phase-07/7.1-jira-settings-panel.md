# Story 7.1 — Jira Connection Settings Panel

## Phase
Phase 7: Jira Integration & Context Binding

## Dependencies
- Story 2.3, 2.5 must be `DONE`.

## Status
`[ ] PENDING`

## Description
Add a Jira configuration form in Workspace Settings to store enterprise Jira connection details securely.

## Acceptance Criteria
- [ ] `jira_configurations` table: `id, workspace_id, jira_url, email, api_token_encrypted, project_keys TEXT[], created_at, updated_at`.
- [ ] `POST /api/workspaces/:id/jira-config` (WORKSPACE_OWNER+): save Jira config. API token encrypted.
- [ ] `GET /api/workspaces/:id/jira-config`: return config with masked token.
- [ ] `POST /api/workspaces/:id/jira-config/test`: test connection via Jira REST API (`GET /rest/api/3/myself`). Returns `{ valid: boolean; accountId: string }`.
- [ ] Frontend: "Jira" tab in Workspace Settings with fields: Jira URL, email, API token (masked), project keys (comma-separated).
- [ ] Multiple project keys supported (e.g., `PROJ`, `INFRA`).
